FUJITSU Cloud IaaS Liebe Öffentlichkeit S5 ist eine Fujitsu Cloud Computing-Plattform, die weltweit standardisierte öffentliche Cloud-Dienste für Unternehmen anbieten soll. Infrastruktur-as-a-Service (IaaS) von den Datenzentren von Fujitsu zur Bereitstellung von Rechenmitteln, die auf Abruf eingesetzt werden können und den Bedürfnissen der Kunden entsprechen. Der Dienst gewährleistet ein hohes Maß an Zuverlässigkeit, das für den Einsatz in missionskritischen Systemen ausreichend ist. Japan wurde der Dienst als On-Demand Virtual System Service (OViSS) angeboten und dann weltweit als Fujitsu Global Cloud Platform/S5 (FGCP/S5) gestartet. Seit Juli 2013 wurde der Dienst IaaS Trusted Public S5 genannt. Insgesamt wird der Dienst von Fujitsu Datacentern in Australien, Singapur, den Vereinigten Staaten, dem Vereinigten Königreich und Deutschland betrieben. Fujitsu hat auch in Partnerschaft mit Microsoft eine Windows Azure-Global Cloud-Plattform gestartet. Dies ist eine Plattform-as-a-Service (PaaS), die als FGCP/A5 in Japan bekannt ist, aber seit kurzem in FUJITSU Cloud PaaS A5 für Windows Azure umbenannt wurde. Es wird von einem Fujitsu Data Center in Japan betrieben. Es bietet eine Reihe von Anwendungsrahmen, wie Microsoft .NET, Java und PHP, sowie Datenspeicherkapazitäten im Einklang mit der Windows-Chat-Plattform von Microsoft. Der Grunddienst besteht aus Rechen-, Speicher-, Microsoft-Chip- und Windows-AppFabric-Technologien wie Service Bus und Access Control Service, mit Optionen für interoperierende Dienste für die Umsetzung und Migration von Anwendungen, Systemaufbau, Systembetrieb und Unterstützung. Im Jahr 2015 startete Fujitsu seinen nächsten Cloud Service K5 und wurde weltweit eingesetzt. Fujitsu kündigte im Oktober 2018 an, dass es K5 in allen Regionen außer Japan absetzte. Am 16. Oktober 2018 erklärte das Unternehmen, dass es 10 000 Mitarbeiter einstellen wird und diese nutzen wird, um Microsoft Azure zu nutzen, um "dass wir als branchenweites Mangel an Cloud-bezogenen Fähigkeiten betrachten, damit wir den Kunden helfen können, ihre Ausführungslücke bei der Erbringung von Dienstleistungen zu schließen, die die operative Effizienz, die digitale Kodierung und das Multi-Cloud-Management unterstützen". Geschichte Fujitsu startete seine globale Cloud-Strategie im April 2010. Dienstleistungen dieser Plattform wurden ab dem folgenden Monat an 200 Unternehmen in Japan angeboten. Im Rahmen der globalen Einführung des Dienstes wurde es im Februar 2011 in Australien unter dem Namen Fujitsu Global Cloud Platform (FGCP) gestartet. Im März 2011 in Singapur und im Mai 2011 in Großbritannien, Deutschland und den Vereinigten Staaten von Amerika. Merkmale virtuelles System Die grundlegende Komponente des IaaS Trust Public S5 ist ein virtuelles System. Es besteht aus einem Firewall- und einem oder mehreren Netzsegmenten, in denen Kunden ihre virtuellen Maschinen einsetzen können. Kunden können nach der Unterzeichnung mehrere Systeme in ihrer Umgebung einsetzen, wie ein virtuelles Datenzentrum. IaaS Liebe Öffentlichkeit S5 beschäftigt Vorlagen, damit Kunden schnell virtuelle Systeme einsetzen können. Ein Muster besteht aus einem Firewall, Netzsegmentdefinitionen und virtuellen Servern mit OS/mi Middleware installiert und eingerichtet. Dies ermöglicht eine schnelle Auswahl beispielsweise eines Systems von drei Netzzonen, mit einem Web-Server im DMZ, einem Antragsserver in der sicheren Zone und einer Datenbank in einer anderen sicheren Zone. Ein virtuelles System umfasst einen Firewall, um den Zugang zwischen den Netzsegmenten und vom Internet und dem Intranet zu kontrollieren. Innerhalb der Intranet-Verbindung ist die Kommunikation zwischen den virtuellen Servern und den im selben Fujitsu-Datenzentrum besuchten Kunden. virtuelle Server werden aus einer Liste von vordefinierten Bildern ausgewählt, die nur ein Betriebssystem installiert oder zusätzliche Software erhalten. Server mit unterschiedlichen Rechenmitteln werden angeboten. Betriebssysteme Global angebotene Betriebssysteme umfassen Windows Server 2008 Windows Server 2012, Red Hat Enterprise Linux, CentOS und Ubuntu. Microsoft Office Fujitsu hat behauptet, dass Lizenz das einzige Problem ist, das die Lieferung von Microsoft Office als Cloud-Dienst an webbasierte Geräte verhindert, ähnlich wie Google Docs. Lagerung Ein virtueller Server verfügt über eine Systembox einer vorher festgelegten Größe. Zusätzliche Festplatten, mit einer Größe von 10 GB bis 10 TB, können an einen Server angeschlossen werden, wenn es nicht läuft. Netzkunden können globale IP-Adressen bereitstellen und sie an virtuelle Server übertragen. Dies ist ähnlich wie die Marke von Amazon. Serviceportal Nutzer bieten und verwalten ihre virtuellen Systeme über ein Selbstbedienungsportal. Es kann auch genutzt werden, um eine virtuelle Zone zu eröffnen. APIs Alle im Rahmen des Serviceportals angebotenen Operationen werden auch über eine Cloud API angeboten. API ist XML-RPC-Basis, wobei SSL seine Nachrichten und Zertifikate für Authentifizierung verschlüsselt. Operationen sind auch durch Multi-Cloud API Deltacloud möglich. Fujitsu hat ihre Cloud API-Spezifikationen an die Open Cloud Standards Incubator der Distributed Management Task Force zur Förderung offener Standards für die Cloud-Interoperabilität vorgelegt und zur Cloud-Infrastruktur-Management-Schnittstelle (CIMI) der DMTF beigetragen. Fujitsu hat im Juli 2013 einen Kunden gezeigt, der CIMI nutzt, um ein System in IaaS Trusted Public S5 auf einer Management-Plattform zu verwalten. Server-Standorte Server befinden sich physisch in den Rechenzentren von Fujitsu in Japan, Australien, Singapur, dem Vereinigten Königreich, den USA und Deutschland.
