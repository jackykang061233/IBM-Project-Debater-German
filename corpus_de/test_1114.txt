Windows RT ist ein von Microsoft entwickeltes deprecated mobiles Betriebssystem. Es ist eine Ausgabe von Windows 8.x für die 32-Bit ARM Architektur (ARMv7.) Erst im Januar 2011 auf der Consumer Electronics Show vorgestellt wurde das Windows RT 8 Betriebssystem offiziell neben Windows 8 am 26. Oktober 2012 gestartet, mit der Veröffentlichung von drei Windows RT-basierten Geräten, einschließlich Microsofts Original Surface Tablet. Im Gegensatz zu Windows 8, Windows RT ist nur als vorinstallierte Software auf Geräten verfügbar, die speziell für das Betriebssystem von Original Equipment Manufacturers (OEMs) entwickelt wurden. Microsoft beabsichtigt für Geräte mit Windows RT nutzen die Leistungsfähigkeit der Architektur, um eine längere Akkulaufzeit zu ermöglichen, System-on-Chip (SoC) Designs verwenden, um dünnere Geräte zu ermöglichen und eine zuverlässige Erfahrung im Laufe der Zeit zu bieten. Im Vergleich zu anderen mobilen Betriebssystemen unterstützt Windows RT auch eine relativ große Anzahl bestehender USB-Peripheriegeräte und Zubehör und umfasst eine für ARM-Geräte optimierte Version von Microsoft Office 2013 als vorinstallierte Software. Während Windows RT jedoch das Aussehen und die Funktionalität von Windows 8 erbt, hat es eine Reihe von Einschränkungen; es kann nur Software ausführen, die digital von Microsoft (die vorinstallierte Software und Windows Store-Apps umfasst), und es fehlt bestimmten Entwickler-orientierten Funktionen. Windows RT wurde veröffentlicht, um gemischte Bewertungen von verschiedenen Steckdosen und Kritikern. Einige fühlten, dass Windows RT-Geräte Vorteile gegenüber anderen mobilen Plattformen (wie iOS oder Android) aufgrund seiner gebündelten Software und der Fähigkeit, eine breitere Vielfalt von USB-Peripheriegeräten und Zubehör zu verwenden, aber die Plattform wurde für ihr schlechtes Software-Ökosystem kritisiert, unter Hinweis auf die frühen Stadium des Windows Store und seine Unvereinbarkeit mit bestehenden Windows-Software, und andere Einschränkungen über Windows 8.Kritiker und Analysten als unklare Funktionen als Windows RT als kommerzielle Einschränkungen, die als Windows als Windows RT-Geräte Einschränkungen betrachtet Verbesserungen an den mobilen Prozessoren von Intel, zusammen mit einer Entscheidung von Microsoft, OEM-Lizenzgebühren für Windows auf Geräten mit Bildschirmen kleiner als 9 Zoll zu entfernen, haben einen Markt für Low-End-Wintel-Tabletten mit der vollen Windows 8-Plattform angestoßen. Diese Geräte sind weitgehend kannibalisiert Windows RT; Anbieter fingen an, ihre Windows RT-Geräte durch schlechte Verkäufe zu löschen, und weniger als ein Jahr nach seiner Veröffentlichung, Microsoft erlitt einen US$900 Millionen Verlust, die weitgehend auf schlechte Verkäufe der ARM-basierten Surface Tablet und unverkauften Aktien verantwortlich war. Nur zwei mehr Windows RT-Geräte, Microsofts Surface 2 und das Nokia Lumia 2520 Ende 2013, wurden über die fünf Original-Start-Geräte veröffentlicht, und kein Windows RT-Gegenstand zum Surface Pro 3 wurde durch eine Neupositionierung der Surface Line in den High-End-Markt freigegeben, und ein Schalter zur Intel-Architektur für die Oberfläche 3. Diese Entwicklungen haben Microsofts zukünftige Unterstützung der Plattform im Zweifelsfall hinterlassen. Mit dem Ende der Produktion für Surface 2 und Lumia 2520 fertigen Microsoft und seine Tochtergesellschaften keine Windows RT-Geräte mehr. Geschichte Auf der Consumer Electronics Show 2011 wurde offiziell bekannt gegeben, dass die nächste Version von Windows die Implementierung von System-on-Chip (SoC) basierend auf der ARM-Architektur unterstützen würde. Steven Sinofsky, dann Windows-Divisionspräsident, demonstrierte eine frühe Version eines Windows-Ports für die Architektur, codierte Windows auf ARM (WoA), die auf Prototypen mit Qualcomm Snapdragon, Texas Instruments OMAP und Nvidia Tegra 2 Chips läuft. Die Prototypen zeigten Arbeitsversionen des Internet Explorer 9 (mit DirectX-Unterstützung über die GPU von Tegra 2), PowerPoint und Word sowie die Verwendung von Klassentreibern, um den Druck auf einen Epson-Drucker zu ermöglichen. Sinofsky fühlte, dass die Verschiebung zu SoC-Designs "eine natürliche Entwicklung von Hardware, die auf eine breite Palette von Formfaktoren anwendbar ist, nicht nur zu Schiefen", während Microsoft CEO Steve Ballmer betonte die Bedeutung der Unterstützung von SoCs auf Windows, indem er erklärte, dass das Betriebssystem " überall auf jedem Gerät ohne Kompromiss sein würde. "Initial development on WoA fand durch die Portierung von Code von Windows 7 statt; Windows Mobile Smartphones wurden verwendet, um frühen Builds von WoA zu testen, weil der Mangel an leicht verfügbaren ARM-basierten Tabletten. Spätere Tests wurden mit einem benutzerdefinierten Array von Rack-montierten ARM-basierten Systemen durchgeführt. Änderungen an der Windows-Codebasis wurden vorgenommen, um das Betriebssystem für die interne Hardware von ARM-Geräten zu optimieren, aber auch eine Reihe von technischen Standards, die traditionell von x86-Systemen verwendet werden. WoA-Geräte würden UEFI-Firmware verwenden und ein softwarebasiertes Trusted Platform Module zur Unterstützung der Geräteverschlüsselung und UEFI Secure Boot haben. ACPI wird auch verwendet, um Plug-and-Play-Geräte zu erkennen und zu steuern und Strommanagement außerhalb der SoC bereitzustellen. Um eine breitere Hardwareunterstützung zu ermöglichen, verwenden Peripheriegeräte wie menschliche Schnittstellengeräte, Speicher und andere Komponenten, die USB- und I2C-Verbindungen verwenden, Klassentreiber und standardisierte Protokolle. Windows Update dient als Mechanismus zur Aktualisierung aller Systemtreiber, Software und Firmware. Microsoft zeigte andere Aspekte des neuen Betriebssystems, die als Windows 8 bekannt sind, während der nachfolgenden Präsentationen. Unter diesen Änderungen (die auch eine überholte Schnittstelle für den Einsatz auf berührungsbasierten Geräten rund um die Metro-Designsprache optimiert enthalten) war die Einführung von Windows Runtime (WinRT). Software, die mit dieser neuen Architektur entwickelt wurde, könnte prozessorunabhängig sein (unter Berücksichtigung der Kompatibilität mit x86- und ARM-basierten Systemen), würde die Verwendung von Touch-Input betonen, würde innerhalb einer sandboxierten Umgebung laufen, um zusätzliche Sicherheit zu bieten und über Windows Store verteilt werden - ein Speicher ähnlich wie die Dienste wie den App Store und Google Play. WinRT wurde auch optimiert, um eine zuverlässigere Erfahrung auf ARM-basierten Geräten zu bieten; als solche, Rückwärtskompatibilität für Win32 Software ansonsten kompatibel mit älteren Versionen von Windows wurde absichtlich von Windows auf ARM ausgeschlossen. Windows-Entwickler gaben an, dass vorhanden Windows-Anwendungen wurden nicht speziell für Zuverlässigkeit und Energieeffizienz auf der ARM-Architektur optimiert und WinRT war ausreichend für die Bereitstellung von "vollen Ausdruckskraft" für Anwendungen, "wenn die Fallen und Fallstricke vermieden werden, die die Gesamterfahrung für die Verbraucher möglicherweise reduzieren können." Konsequenziell würde diese mangelnde Rückwärtskompatibilität auch verhindern, dass bestehende Malware auf dem Betriebssystem läuft. Am 16. April 2012 kündigte Microsoft an, dass Windows auf ARM offiziell als Windows RT bezeichnet wird. Microsoft hat nicht explizit angezeigt, was die RT im Namen des Betriebssystems erwähnt, aber es wurde angenommen, auf die WinRT-Architektur zu verweisen. Steven Sinofsky erklärte, dass Microsoft sicherstellen würde, dass die Unterschiede zwischen Windows RT und 8 in der Werbung angemessen angesprochen wurden. Berichte fanden jedoch heraus, dass Werbe-Webseiten für die Microsoft Surface-Tablette verwirrende Formulierungen enthalten, die auf die Kompatibilitätsunterschiede hindeuten und dass Microsoft Store-Vertreter uneinheitliche und manchmal falsche Informationen über Windows RT bereitstellen. Als Reaktion, Microsoft sagte, dass Microsoft Store-Mitarbeiter würden durchschnittlich 15 Stunden Training vor der Einführung von Windows 8 und Windows RT gegeben werden, um sicherzustellen, dass die Verbraucher in der Lage, die richtige Wahl für ihre Bedürfnisse zu treffen. Die ersten Windows RT-Geräte wurden offiziell neben Windows 8 am 26. Oktober 2012 veröffentlicht. Windows 8.1, ein Upgrade für Windows 8 und RT, wurde im Windows Store am 17. Oktober 2013 veröffentlicht, mit einer Reihe von Verbesserungen an der Schnittstelle und Funktionalität des Betriebssystems. Für Windows RT-Geräte fügt das Update auch Outlook in die enthaltene Office RT-Suite. Das Update wurde von Microsoft kurz nach seiner Veröffentlichung vorübergehend zurückgewiesen, nachdem einige Surface-Benutzer einen seltenen Fehler aufgetreten hatten, der die Boot-Konfigurationsdaten des Geräts während der Installation beschädigte, was zu einem Fehler beim Start führte. Am 21. Oktober 2013 veröffentlichte Microsoft Recovery-Medien und Anweisungen, die verwendet werden könnten, um das Gerät zu reparieren und den Zugriff auf Windows 8.1 am nächsten Tag wiederherzustellen. Unterschiede von Windows 8 Während Windows RT ähnlich wie Windows 8 funktioniert, gibt es noch einige bemerkenswerte Unterschiede, vor allem mit Software und Hardware-Kompatibilität. Julie Larson-Green, dann Executive Vice President der Devices and Studios Gruppe bei Microsoft, erklärte, dass Windows RT wurde letztlich entworfen, um eine "geschlossene, schlüsselfertige" Benutzererfahrung zu bieten, "wo es nicht alle Flexibilität von Windows hat, aber es hat die Macht von Office und dann alle neuen Stil Anwendungen. Also könnten Sie es Ihrem Kind geben und er wird es nicht mit einem Haufen von Symbolleisten versehentlich aus dem Internet Explorer laden und dann später zu Ihnen kommen und sagen: "Warum bekomme ich all diese Pop-ups? ' Es ist einfach nicht in der Lage, das durch Design zu tun." Inklusive Software Windows RT beinhaltet nicht Windows Media Player, zugunsten anderer Multimedia-Apps, die im Windows Store gefunden werden; Geräte werden mit den internen Xbox Music und Xbox Video-Apps vorgeladen. Alle Windows RT-Geräte umfassen Office 2013 Home & Student RT – eine Version von Microsoft Office, die für ARM-Systeme optimiert ist. Da die Version von Office RT auf Windows RT-Geräten basiert auf der Home & Student-Version, kann es nicht für "kommerzielle, gemeinnützige oder Umsatz generierende Aktivitäten" verwendet werden, es sei denn, die Organisation hat eine Volumenlizenz für Office 2013, oder der Benutzer hat ein Office 365 Abonnement mit kommerziellen Nutzungsrechten. Aus Kompatibilitäts- und Sicherheitsgründen sind bestimmte erweiterte Funktionen wie Visual Basic Makros nicht in Office RT verfügbar. Windows RT beinhaltet auch ein BitLocker-basiertes Geräteverschlüsselungssystem, das die Daten eines Benutzers passiv verschlüsselt, sobald sie sich mit einem Microsoft-Konto anmelden. Software-Kompatibilität Aufgrund der unterschiedlichen Architektur von ARM-basierten Geräten im Vergleich zu x86 Geräten hat Windows RT Software-Kompatibilitätsbeschränkungen. Obwohl das Betriebssystem neben der berührungsorientierten Benutzeroberfläche von Windows 8 noch die traditionelle Windows-Desktop-Umgebung bietet, sind die einzigen Desktop-Anwendungen, die von Windows RT offiziell unterstützt werden, diejenigen, die mit dem Betriebssystem selbst kommen; wie File Explorer, Internet Explorer und Office RT. Nur Windows Store-Apps können von Benutzern auf Windows RT-Geräten installiert werden; sie müssen aus Windows Store oder Sideloaded in Unternehmensumgebungen. Entwickler können Desktop-Anwendungen nicht portieren, um unter Windows RT laufen, da Microsoft-Entwickler fühlten, dass sie nicht richtig für die Plattform optimiert werden. Folglich unterstützt Windows RT auch nicht "new-experience aktiviert" Web-Browser: eine spezielle Klasse von App unter Windows 8 verwendet, die Web-Browser erlaubt, Varianten zu bündeln, die in der Windows RT "moderne Benutzeroberfläche" laufen können und in andere Apps integrieren, aber immer noch Win32-Code wie Desktop-Programme verwenden. Hardware-Kompatibilität In einer Präsentation bei Windows 8 Start-Event in New York City behauptet Steven Sinofsky, dass Windows RT 420 Millionen bestehende Hardware-Geräte und Peripheriegeräte unterstützen würde. Im Vergleich zu Windows 8 steht jedoch keine volle Funktionalität für alle Geräte zur Verfügung, und einige Geräte werden überhaupt nicht unterstützt. Microsoft bietet ein "Compatibility Center"-Portal, in dem Benutzer nach Kompatibilitätsinformationen auf Geräten mit Windows RT suchen können; auf Start, die Website aufgelistet nur über 30.000 Geräte, die mit dem Betriebssystem kompatibel waren. Netzwerk- und Gerätemanagement Während Windows RT-Geräte können einer HomeGroup beitreten und auf Dateien zugreifen, die in freigegebenen Ordnern und Bibliotheken auf anderen Geräten innerhalb der Gruppe gespeichert sind, können Dateien nicht vom Windows RT-Gerät selbst geteilt werden. Windows RT unterstützt nicht die Verbindung zu einer Domain für Netzwerk-Logins, noch unterstützt es die Verwendung von Group Policy für Gerätemanagement. Exchange ActiveSync, der Windows Intune-Service oder System Center Configuration Manager 2012 SP1 können jedoch verwendet werden, um einige Kontrolle über Windows RT-Geräte in Unternehmensumgebungen, wie die Fähigkeit, Sicherheitsrichtlinien anzuwenden und ein Portal zu bieten, das verwendet werden kann, um Anwendungen von außerhalb Windows Store. Support Lifecycle Im Gegensatz zu Windows 8 (die, pro Standardrichtlinien für Windows-Versionen, erhält rund 5 Jahre Mainstream-Unterstützung,) Microsoft hat kein bestimmtes Datum für das Ende der Mainstream-Unterstützung für Windows RT bekannt gegeben. Das Original Surface Tablet fiel unter Microsofts Support-Policies für Consumer Hardware und erhielt Mainstream-Unterstützung bis 11. April 2017. Geräte Microsoft verhängte eine enge Kontrolle über die Entwicklung und Produktion von Windows RT-Geräten: Sie wurden in Zusammenarbeit mit dem Unternehmen entworfen und auf strenge Design- und Hardware-Spezifikationen gebaut, einschließlich Anforderungen nur zugelassene Modelle bestimmter Komponenten zu verwenden. Jailbreak Exploit Im Januar 2013 wurde im Windows-Kernel ein Privileg escalation-Exploit entdeckt, mit dem unsigned-Code unter Windows RT ausgeführt werden kann; der Exploit beinhaltete die Verwendung eines Remote-Debugging-Tools (von Microsoft bereitgestellt, um WinRT-Apps auf Windows RT-Geräten zu debuggen), um Code auszuführen, der die im RAM gespeicherte Signing-Ebene ändert, um unbesigned-Code ausführen zu können (standardt, es wird auf eine Ebene gesetzt, die nur den Code ausführen kann von Microsoft unterzeichneten Code ausführen). Neben seiner Erklärung des Exploits, beinhaltete der Entwickler auch einen persönlichen Appell an Microsoft urging sie, um die Einschränkungen auf Windows RT-Geräte zu entfernen, vorausgesetzt, dass ihre Entscheidung nicht aus technischen Gründen, und dass die Geräte wären wertvoller, wenn diese Funktionalität zur Verfügung stehen. In einer Erklärung applaudierte ein Microsoft-Sprecher den Aufwand, was darauf hindeutete, dass der Exploit keine Sicherheitsbedrohung darstellt, weil er administrativen Zugriff auf das Gerät, fortgeschrittene Techniken erfordert und trotzdem Programme für ARM neu kompiliert werden müssten. Microsoft hat jedoch noch angezeigt, dass der Exploit in einem zukünftigen Update gepatelt werden würde. DemiseIm November 2013 sprach Julie Larson-Green über Windows RT auf der UBS Global Technology Conference, über die Zukunft der Microsoft-Mobilstrategie rund um die Windows-Plattform. Larson-Green sagte, dass in der Zukunft (Berechnung für Windows, Windows RT und Windows Phone,) Microsoft war "[nicht] würde drei [mobile Betriebssysteme haben." Das Schicksal von Windows RT wurde durch ihre Bemerkungen unklar gelassen; Industrieanalysten interpretierten sie als Zeichen, dass Microsoft bereit war, Windows RT aufgrund seiner schlechten Annahme einzustellen, während andere schlugen vor, dass Microsoft Windows mit Windows Phone zu vereinen. Microsoft kündigte schließlich seine "Universal Windows Apps"-Plattform auf Build 2014 an, die es Entwicklern ermöglichen würde, WinRT-Apps für Windows, Windows Phone und Xbox One zu erstellen, die gemeinsame Codebases teilen. Diese Initiativen wurden von einem Ziel für Windows 10 zusammengeführt, um das Kern Windows-Betriebssystem auf allen Geräten zu vereinheitlichen. Die Kritik interpretierte Microsofts Umzug, um die Einführung eines kleineren Surface-Modells im Mai 2014 als ein weiteres Zeichen, dass Microsoft, unter neuem CEO Satya Nadella, und neuer Gerätekopf Stephen Elop (die Microsoft beim Kauf von Nokias Handy-Geschäft im September 2013, nur um das Unternehmen im folgenden Jahr verlassen,) plante, weiter herunterzuspielen Windows RT, da das Unternehmen RT seine Aufmerksamkeit auf eine höhere, produktivitätsorientierte Markt mit dem Propass verschoben hatte. Analysten glaubten, dass Microsoft plante, seinen Erwerb von Nokias Gerätegeschäft für zukünftige Windows RT-Geräte zu nutzen, möglicherweise unter der Marke Lumia; dies stellte sich letztlich als ein Ausfall heraus, und Microsoft würde schließlich verlassen den Verbraucher Handy-Markt, verkauft seine Vermögenswerte an Foxconn und HMD Global im Mai 2016. Neue Intel Prozessoren für mobile Geräte waren im Vergleich zu ARM-Äquivalenten in Bezug auf Leistung und Akku-Lebensdauer wettbewerbsfähiger; dieser Faktor und andere Änderungen von Microsoft, wie die Entfernung von Windows OEM-Lizenzgebühren auf Geräten mit Bildschirmen weniger als 9 Zoll in der Größe, hat die Schaffung eines Marktes für Low-End-Tabletten mit dem vollen Windows 8-Betriebssystem auf Intel-kompatiblen Plattformen, so dass weitere Unsicherheit über Microsofts Unterstützung von ARMbi außerhalb von Smartphones bleiben. Ein solches Gerät kam im März 2015, als Microsoft ein neues Low-End-Oberflächenmodell enthüllte, die Intel Atom-basierte Oberfläche 3; im Gegensatz zu früheren Low-End-Oberflächenmodellen, Surface 3 nicht verwendet ARM und Windows RT. Im Juni 2016 kündigte Microsoft an, dass die Produktion dieses Geräts bis Dezember 2016 enden würde, wobei der Umsatz im folgenden Monat endet. In diesem Segment wurde kein Nachfolgegerät geplant, das den Abgang des Unternehmens vom Low-End-Tablettenmarkt für Windows-Verbraucher signalisiert, wo Experten ständig darüber diskutieren, ob eine Zukunft existiert. Dezember 2016 kündigte Microsoft an, dass es im Rahmen einer Partnerschaft mit Qualcomm eine ARM-Version von Windows 10 für Snapdragon-basierte Geräte starten sollte, die sich zunächst auf Laptops konzentriert. Im Gegensatz zu Windows RT ermöglicht die ARM-Version von Windows 10 die Verwendung eines x86 Prozessor-Emulators, Win32 Desktop-Software auszuführen, anstatt nur Apps aus Windows Store zuzulassen. Im folgenden Jahr kündigte Microsoft die Marke Always Connected PC an, die Windows 10-Geräte mit zellulärer Konnektivität umfasste; der Start verfügte über zwei Snapdragon 835-powered 2-in-1 Laptops von Asus und HP und eine Integration von Qualcomm's Snapdragon X16 Gigabit LTE Modem mit AMD Ryzen Mobile Plattform. Am 2. Mai 2017 enthüllte Microsoft Windows 10 S, eine Ausgabe von Windows 10, die vor allem für Low-End-Mobilgeräte entwickelt wurde, die auf den Bildungsmarkt ausgerichtet sind (vor allem mit Googles Linux-basiertem Chrome OS). Ähnlich wie bei Windows RT, es beschränkt Software-Installation auf Anwendungen, die über Windows Store erhalten.
