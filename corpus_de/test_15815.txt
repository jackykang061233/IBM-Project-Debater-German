Ein mobiler Browser ist ein Webbrowser, der für den Einsatz auf einem mobilen Gerät wie einem Mobiltelefon oder einem Smartphone bestimmt ist. Mobile Browser werden optimiert, um zu zeigen Webinhalte am wirksamsten für kleine Bildschirme auf tragbaren Geräten. Mobile Browser-Software muss klein und effizient sein, um die geringe Speicherkapazität und die geringe Bandbreite an drahtlosen Endgeräten zu decken. In der Regel wurden sie jedoch Web-Browser ablösen, aber einige neuere mobile Browser können auch neueste Technologien wie CSS 3, JavaScript und Audi bedienen. Websites, die so ausgelegt sind, dass sie von diesen Browsern genutzt werden können, werden als drahtlose Portale oder gemeinsam als Mobile Web bezeichnet. Sie können automatisch eine mobile Version jeder Seite erstellen, z.B. auf der Website von Wikipedia. Basistechnologie Der mobile Browser verbindet in der Regel über ein zelluläres Netz oder zunehmend über Wireless LAN, wobei Standard-Grenzwert über das iPhone/IP verwendet und Web-Seiten auf der Website von HTML, XHTML Mobile Profile (WAP 2.0) oder WML (die sich aus der HDML entwickelt). WML und HDML sind gebündelte Formate, die für die Übertragung über eine begrenzte Bandbreite geeignet sind, und drahtlose Datenverbindung namens WAP. DoCoMo hat in Japan den i-mode-Dienst definiert, der auf i-mode-Computern basiert, die eine Erweiterung des Compact-Computers (C-HTML) ist, einem einfachen Teil von HTML. WAP 2.0 definiert XHTML Mobile Profile plus WAP CSS, Untersets des Standards XHTML und CSS mit geringfügigen mobilen Erweiterungen. Neue mobile Browser sind voll ausgestaltete Web-Browser, die in der Lage sind, aus dem Computer, dem CSS, ECMAScript, sowie mobilen Technologien wie WML, i-mode-Computer oder cHTML zu bestehen. Um kleine Bildschirme zu erfassen, verwenden sie die Schnittstellen Post-WIMP.
