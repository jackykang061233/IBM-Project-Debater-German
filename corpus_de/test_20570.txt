HP Cloud war eine Reihe von Cloud-Computing-Services, die von Hewlett-Packard (HP) angeboten wurden, die Public Cloud, Private Cloud, Hybrid Cloud, Managed Private Cloud und andere Cloud-Services anbieten. Es war die Kombination aus der bisherigen HP Converged Cloud Business Unit und HP Cloud Services, die die OpenStack Technologie-basierte Public Cloud ist. Es wurde für Unternehmen vermarktet, öffentliche Cloud-Dienste mit internen IT-Ressourcen zu kombinieren, um Hybrid Clouds zu schaffen, oder eine Mischung aus privaten und öffentlichen Cloud-Umgebungen, von 2011 bis 2016. Geschichte HP Converged Cloud wurde im April 2012 bekannt gegeben. HP Converged Cloud wird unter einer Geschäftseinheit Hewlett-Packard verwaltet, die 2013 benannt wurde Die Converged Cloud-Einheit, geleitet von Saar Gillai als Senior Vice President und General Manager von Converged Cloud. HP Public Cloud wurde am 14. März 2011 bekannt gegeben und als öffentliche Beta am 10. Mai 2012 gestartet. HP Fellow und MySQL Autor Brian Aker kündigten den Relational Database Service auf der Bühne auf der 2012 MySQL User's Conference. Die HP Public Cloud Beta, die im Mai 2012 live ging, umfasste Komponenten des OpenStack-Technologie-basierten Speicher- und Content-Liefernetzwerks (CDN). HP Cloud Object Storage und HP Cloud CDN wurden am 1. August 2012 in die allgemeine Verfügbarkeit überführt. HP Cloud DNS als Service wurde am 1. Juli 2013 in die allgemeine Verfügbarkeit gebracht. Die beiden Geschäftseinheiten wurden Ende 2013 zusammengeführt und bei der HP Discover Veranstaltung in Barcelona, Spanien, bekannt gegeben. Am 7. Mai 2014 kündigte HP das HP Helion-Portfolio von Produkten und Dienstleistungen an und erklärte, dass das Unternehmen plante, mehr als 1 Milliarde in Cloud zu investieren. HP Helion umfasste die bestehenden Cloud-Produkte von HP, neue, auf OpenStack-Technologie basierende Services sowie professionelle und Support-Dienste, um Unternehmen bei der Entwicklung und Verwaltung von hybriden IT-Umgebungen zu unterstützen. Am 21. Oktober 2015 gab HP bekannt, dass es die HP Cloud im Januar 2016 abschalten würde. Cloud Computing-Herausforderungen Wechseln zwischen Clouds Business-Organisationen, die Cloud Computing nutzen möchten, haben die Wahl, eine private Cloud zu verwenden, die eine Cloud-Infrastruktur ist, die speziell für eine einzelne Organisation läuft, die sie selbst oder von einem Drittanbieter gebaut hat, oder eine Public Cloud, wobei ein Dienstleister Anwendungen, Storage und andere Ressourcen der Allgemeinheit zur Verfügung stellt. Die Entscheidung, die zu verwenden ist, basiert auf einer Reihe von Faktoren, wie zum Beispiel, ob das Unternehmen in einem stark regulierten Sektor tätig ist, wie die Pharmaindustrie, die Vorschriften über die Kontrolle und Sicherheit von Daten einhalten muss, oder wenn das Unternehmen Dienstleistungen schnell in den Markt bringen muss, wie ein Web-Anwendungen-Entwickler. Die erste würde sich für eine private Cloud für den Sicherheitsfrieden des Geistes entscheiden, während der Web-Entwickler würde eine öffentliche Cloud-Angebot wählen. Die meisten Unternehmen würden eine Mischung der beiden betreiben, auch als Hybrid Cloud-Lieferung bekannt. Mithilfe einer Hybrid-Liefer Cloud-Strategie können Unternehmen verschiedene Cloud-Lieferdienste für bestimmte Anwendungen nutzen. Eine öffentliche Cloud kann beispielsweise ein kostengünstigererer Service für die rechnerintensive Aufgabe der Analyse sein, die Daten würden jedoch lokal oder privat bleiben, um die Vorschriften einzuhalten. Da durch eine Hybrid-Lieferstrategie Unternehmen Anwendungen und Dienstleistungen über verschiedene Clouds, Collokation und Rechenzentren ausführen können, ist ein Vorteil eines solchen Liefermechanismus, dass Organisationen eine App von einem geographischen Standort in einen anderen in großen Stürmen bewegen können. Für bestimmte Aufgaben können verschiedene Arten von Hybrid-Wolken auftreten. Das Infrastructure as a service (IaaS)-Modell kann eine Hybrid-Cloud sein, indem es Betriebssysteme und Virtualisierungstechnologien sowohl von einer öffentlichen als auch privaten Cloud-Infrastruktur nutzen könnte, um Cloud-Services anzubieten. Weitere Modelle, die die Cloud mit öffentlichen Cloud-Technologien mischen, sind Platform as a service (PaaS), die Infrastrukturdienste in Verbindung mit Entwicklungstools und Middleware umfasst, und Software als Service (SaaS), der zur Entwicklung von Paketsoftware verwendet wird. Bewegliche Daten in der Cloud können jedoch schwierig sein. Bei Software als Service (SaaS) ist es insbesondere schwierig, wenn der incumbent Cloud-Provider proprietäre Software verwendet oder eine Open Source-Anwendung verändert hat. Eine Möglichkeit, Cloud-Migrationsschwierigkeiten zu mindern, ist die Anwendung von Architekten für die Cloud, die die Abhängigkeiten zwischen dem Cloud-Anwendungsstapel und den Fähigkeiten des Cloud-Dienstleisters reduzieren oder beseitigen. Eine andere Möglichkeit besteht darin, nur generische und hochrangige Anwendungen auszuwählen, um sich an erster Stelle in die Cloud zu bewegen.Eine weitere Methode ist die Auswahl offener Standards für Cloud Computing. Eine beste Praxis zur Vermeidung von Cloud-Migrationsproblemen ist die Verwendung einer Management-Plattform, die Anwendungen in jeder Umgebung unterstützen kann, die eine Benutzerorganisation verwenden kann. Zum Beispiel können Cloud-Experten empfehlen, dass Unternehmen, die an Kosteneinsparungen interessiert sind, geeignete Anwendungen in der Public Cloud platzieren, aber eine höhere Sicherheitsanwendung in einer privaten Cloud oder Hybrid Cloud behalten. In einem 2013 wettbewerbsfähigen Hersteller-Analysebericht der International Data Corporation, einem Marktforschungsunternehmen, geben die Autoren an, dass Cloud-Profi-Dienste Unternehmen dabei unterstützen können, eine kurzfristige und langfristige Strategie für die Nutzung von Cloud in einer Weise zu erstellen, die ihre Geschäftsziele unterstützt. Eine Virtual Private Cloud (VPC) besteht aus einer Sammlung von Computing-Ressourcen, die innerhalb der Multitenant-Architektur einer Public Cloud-Installation geteilt und zugewiesen werden. Der VPC kann auf Anfrage konfiguriert werden und ermöglicht es einem VPC-Benutzer, von anderen VPC-Nutzern isoliert zu werden. HP Helion Managed Virtual Private Cloud ist eine verwaltete Cloud mit VPC-Funktionen. Es umfasst Sicherheitsfeatures für Unternehmen, die über ein Vor-Ort-Netzwerk mit einer Public Cloud verbunden sind, mit Software-definierter Netzwerktechnologie. Es gibt fünf verschiedene Größen von VPC-Installationen. Unternehmen können entweder die VPC selbst verwalten oder eine Gebühr zahlen, um HP verwalten. Mehrere Kopien eines Virtual LAN (VLAN) werden mit HP Helion Managed Virtual Private Cloud geliefert. Dies bietet eine Methode für Unternehmen, über die Firewall von ihrem Vor-Ort-Netzwerk in den VPC zu verbinden, so dass der VPC Teil des Vor-Ort-Netzwerks erscheint. Unternehmen, die HP Helion Managed Virtual Private Cloud nutzen, erhalten täglich verschlüsselte Backups, die von HP durchgeführt werden. Der Dienst führt auch automatisierten Lastausgleich über die virtuellen Server in der Cloud durch. Daten werden für die File-Systeme und Server-Betriebssysteme außerhalb der Site gespeichert. Benutzerorganisationen wählen, diese Daten von HP für 30, 60 oder 90 Tage zu speichern. HP behält lokale Backups von SQL Server oder Oracle-Datenbanken im HP-Datenzentrum für 15 oder 30 Tage bei; Offsite-Backups können 15 bis 90 Tage aufbewahrt werden. Benutzerorganisationen wählen 50GB, 100GB oder ein-Terabyte-Blöcke Speicherplatz Netzwerk (SAN) Speicher für Server-Knoten, für die hochverfügbare Clustering hinzugefügt werden kann.
