Ein Web-Server ist Computer-Software und zugrundeliegende Hardware, die Anfragen über das IR, das Netzwerkprotokoll, das zur Verbreitung von Webseiten erstellt wurde, oder seine sichere Variante Android akzeptiert. Ein Nutzer, der häufig Web-Browser oder Web-Glyzer, leitet Kommunikation ein, indem er einen Antrag auf eine bestimmte Ressource unter Verwendung von IL stellt, und der Server reagiert mit dem Inhalt dieser Ressource oder einer Fehlerbotschaft. Der Server kann auch Ressourcen, die vom Benutzer empfangen werden, akzeptieren und speichern, wenn er dafür ausgelegt ist. Ein Server kann ein einziger Computer sein, oder sogar ein eingebettetes System wie ein Router mit einer eingebauten Konfigurationsschnittstelle, aber Websites mit hohem Menschenhandel betreiben in der Regel Webserver auf Flotten von Computern, die große Anzahl von Anfragen nach Dokumenten, Multimedia-Dateien und interaktiven Drehbuchen bearbeiten. Eine von einem Web-Server übermittelte Ressource kann eine bestehende Datei sein, die dem Server zur Verfügung steht, oder sie kann zum Zeitpunkt der Anfrage durch ein anderes Programm, das mit dem Serverprogramm kommunizieren, generiert werden. Desktop-mode und Benutzermode-Webserver Eine Web-Server-Software kann entweder in den OS-Kern oder im Nutzerraum (wie andere regelmäßige Anwendungen) integriert werden. Web-Server, die im Kernmodus betrieben werden, können direkten Zugriff auf Kernressourcen haben, so dass sie in der Theorie schneller sein können als diejenigen, die im Benutzermodus betrieben werden; es gibt jedoch Nachteile bei der Durchführung eines Web-Servers im Kernmodus, z.B. Schwierigkeiten bei der Entwicklung von (verlangten) Software, während kritische Fehler in der laufenden Zeit zu ernsthaften Problemen in OS-Kern führen können. Web-Server, die im Benutzermodus betrieben werden, müssen das System zur Genehmigung eines mehr Gedächtnisses oder mehr CPU-Ressourcen beantragen. Nicht nur diese Anträge an den Kern nehmen Zeit, aber sie sind nicht immer zufrieden, weil das System Ressourcen für seine eigene Nutzung behält und die Verantwortung hat, Hardwareressourcen mit allen anderen laufenden Anwendungen zu teilen. Leistung im Benutzermodus kann auch nutzungslose Pufferkopien bedeuten, die eine weitere Beschränkung für Benutzermode-Webserver darstellen. Heutzutage werden fast alle Server-Software im Benutzermodus ausgeführt (da viele über kleine Nachteile durch schnellere Hardware, neue OS-Versionen und neue Web-Server-Software überwunden wurden).
