Installation (oder Einrichtung) eines Computerprogramms (einschließlich Gerätetreiber und Plugins) ist der Akt, das Programm zur Ausführung bereit zu stellen. Die Installation bezieht sich auf die jeweilige Konfiguration einer Software oder Hardware, um sie mit dem Computer nutzbar zu machen. Für die Installation ist eine weiche oder digitale Kopie des Softwarestücks (Programm) erforderlich. Es gibt verschiedene Prozesse der Installation eines Stücks von Software (Programm). Da der Prozess für jedes Programm und jeden Computer variiert, kommen Programme (einschließlich Betriebssysteme) oft mit einem Installer, ein spezialisiertes Programm, das für das Tun verantwortlich ist, was nötig ist (siehe unten) für die Installation. Die Installation kann Teil eines größeren Software-Bereitstellungsprozesses sein. Die Installation beinhaltet typischerweise, dass Code (Programm) von den Installationsdateien kopiert/generiert wird, um neue Dateien auf dem lokalen Computer für einen einfacheren Zugriff durch das Betriebssystem, die Erstellung von notwendigen Verzeichnissen, die Registrierung von Umgebungsvariablen, die Bereitstellung von separaten Programm für die Deinstallation usw. Da Code in der Regel kopiert/generiert wird in mehreren Standorten, Entinstallation in der Regel mehr als nur das Löschen des Programmordners. Beispielsweise müssen Registrierungsdateien und andere Systemcodes für eine vollständige Deinstallation geändert oder gelöscht werden. Einige Computerprogramme können ausgeführt werden, indem sie einfach in einen auf einem Computer gespeicherten Ordner kopiert und ausgeführt werden. Andere Programme werden in einer für eine sofortige Ausführung ungeeigneten Form geliefert und benötigen daher einen Installationsvorgang. Nach der Installation kann das Programm immer wieder ausgeführt werden, ohne dass vor jeder Ausführung neu installiert werden muss. Zu den gemeinsamen Operationen während der Softwareinstallation gehören: Stellen Sie sicher, dass die erforderlichen Systemanforderungen erfüllt sind Überprüfungen für bestehende Versionen der Software Erstellen oder Aktualisierung von Programmdateien und Ordnern Hinzufügen von Konfigurationsdaten wie Konfigurationsdateien, Windows-Registryeinträge oder Umgebungsvariablen Die Software für den Benutzer zugänglich zu machen, z.B. indem Sie Links, Shortcuts oder Bookmarks erstellen Konfigurieren von Komponenten, die automatisch ausgeführt werden, wie z.B. Daemons oder Windows-Dienste Produktaktivierung durchführen Diese Operationen können einige Gebühren verlangen oder kostenlos sein. Bei der Zahlung sind die Kosten für die Installation, die durch die Installation der Fahrer oder der Ausrüstung in den Räumlichkeiten der Kunden verbunden und relevant oder entstehen. Einige Installateure können versuchen, Benutzer in der Installation von Junkware wie verschiedene Formen von Adware, Toolbars, Tributware oder Software von Partnering-Unternehmen zu trickn. Um dies zu verhindern, ist eine zusätzliche Vorsicht auf das, was genau gefragt wird, installiert zu werden, erforderlich. Die Installation zusätzlicher Software kann dann einfach übersprungen oder unkontrolliert werden (dies kann den Benutzer dazu verpflichten, die benutzerdefinierte, detaillierte oder kompetente Version des Installationsverfahrens zu verwenden). Ein solches bösartiges Verhalten ist nicht unbedingt eine Entscheidung der Softwareentwickler oder deren Unternehmen, sondern kann auch eine Frage externer Installateure wie dem Download.com Installer von CNET sein. Zum Beispiel Commodore veröffentlichte den Installer für Amiga. Eine Klasse moderner Anwendungen, die keine Installation benötigen, sind als tragbare Anwendungen bekannt, da sie auf verschiedenen Computern herumgerömmt und laufen können. Ebenso gibt es Live-Betriebssysteme, die keine Installation benötigen und direkt von einer bootfähigen CD, DVD, USB-Flash-Laufwerk oder über das Netzwerk geladen werden können, wie bei dünnen Clients. Ein Installationsprozess benötigt normalerweise einen Benutzer, der ihn besucht, um Entscheidungen zu treffen, wie die Annahme oder Absenkung eines Endbenutzer-Lizenzvertrags (EULA), die Präferenzen wie den Installationsort, die Bereitstellung von Passwörtern oder die Unterstützung bei der Produktaktivierung. In grafischen Umgebungen sind Installateure, die eine Assistent-basierte Schnittstelle anbieten, häufig. Attended Installer können Benutzer bitten, die Fehler zu mildern. Wenn beispielsweise die Festplatte, in der das Computerprogramm installiert ist, voll ist, kann der Installer den Benutzer bitten, einen anderen Zielpfad anzugeben oder genügend Platz in der Festplatte zu löschen. Ein häufiger Missverständnis ist eine Unarchivierung, die nicht als Installationsaktion betrachtet wird, weil sie keine Nutzerwahlen umfasst, wie die Annahme oder die Absenkung der EULA. Stille Installation Installation Installation, die keine Nachrichten oder Fenster während ihres Fortschritts zeigt."Stille Installation" ist nicht die gleiche wie "unverzichtte Installation" (siehe unten:) Alle stillen Installationen sind unbeaufsichtigt, aber nicht alle unbeaufsichtigten Installationen sind still. Der Grund für eine stille Installation kann Bequemlichkeit oder Unterwäsche sein. Malware ist fast immer still installiert. Für normale Benutzer ist die stille Installation nicht viel zu verwenden, aber in größeren Organisationen, wo Tausende von Benutzern arbeiten, die Bereitstellung der Anwendungen wird eine typische Aufgabe und aus diesem Grund wird die stille Installation durchgeführt, so dass die Anwendung im Hintergrund installiert wird, ohne die Arbeit des Benutzers zu beeinflussen. Unbeaufsichtigte Installation, die ohne Benutzerinteraktion während seines Fortschritts oder ohne Benutzer vorhanden durchgeführt wird. Eines der Gründe für diesen Ansatz ist die Automatisierung der Installation einer Vielzahl von Systemen. Eine unbeaufsichtigte Installation erfordert entweder nicht, dass der Benutzer etwas liefert oder alle notwendigen Eingaben vor dem Start der Installation erhalten hat. Ein solcher Eingang kann in Form von Befehlszeilenschaltern oder einer Antwortdatei, einer Datei sein, die alle notwendigen Parameter enthält. Windows XP und die meisten Linux Distributionen sind Beispiele für Betriebssysteme, die mit einer Antwortdatei installiert werden können. Bei unbeaufsichtigter Installation wird davon ausgegangen, dass es keinen Benutzer gibt, um Fehler zu mindern. Wenn das Installationsmedium beispielsweise fehlerhaft war, sollte der Installer die Installation aussagen, da es keinen Benutzer gibt, um den Fehler zu beheben oder das Medium zu ersetzen. Da eine kopflose Installation keinen Benutzer am Ort des Zielcomputers benötigt, können unbeaufsichtigte kopflose Installateure dazu verwendet werden, ein Programm auf mehreren Maschinen gleichzeitig zu installieren. Geplante oder automatisierte Installation Ein Installationsprozess, der auf einer vorgegebenen Zeit läuft oder wenn eine vorgegebene Bedingung transpires, im Gegensatz zu einem Installationsprozess, der explizit auf dem Befehl eines Benutzers beginnt. So kann beispielsweise ein Systemadministrator, der bereit ist, eine spätere Version eines Computerprogramms zu installieren, das verwendet wird, diese Installation planen, wenn dieses Programm nicht läuft. Eine saubere Installation ist eine, die in Abwesenheit von störenden Elementen wie alten Versionen des Computerprogramms installiert oder Überreste von einer vorherigen Installation durchgeführt wird. Dies kann einfach eine Kopie der ursprünglichen Medien sein, aber Software-Verleger, die Website-Lizenzen für institutionellen Kunden bieten kann eine Version für die Installation über ein Netzwerk.
