o Virt ist eine kostenlose, Open-Source Virtualisierungs-Management-Plattform. Es wurde von Red Hat als Gemeinschaftsprojekt gegründet, auf dem Red Hat Enterprise Virtualization basiert. Es ermöglicht eine zentralisierte Verwaltung von virtuellen Maschinen, Rechen-, Speicher- und Netzwerkressourcen, von einem einfach zu bedienenden webbasierten Frontend mit plattformunabhängigem Zugriff. KVM auf x86-64 und PowerPC64-Architektur sind die einzigen Hypervisoren unterstützt, aber es gibt einen kontinuierlichen Aufwand, die ARM-Architektur in den zukünftigen Versionen zu unterstützen. Architektur oVirt besteht aus zwei Grundkomponenten, oVirt Motor und oVirt Knoten. Das oVirt-Motor-Backend ist in Java geschrieben, während das Frontend mit GWT Web-Toolkit entwickelt wird. Die oVirt-Engine läuft auf dem Applikationsserver WildFly (früher JBoss). Das Frontend kann über ein Webadmin-Portal für die Verwaltung oder ein Benutzerportal mit Privilegien und Funktionen, die fein abgestimmt werden können, aufgerufen werden. Die Benutzerverwaltung kann lokal oder durch die Integration von oVirt mit LDAP- oder AD-Diensten verwaltet werden. Die oVirt-Engine speichert Daten in einer PostgreSQL-Datenbank. Daten-Warehousing und Reporting-Fähigkeiten hängen von zusätzlichen Historien ab und meldet Datenbanken, die während des Setup-Verfahrens optional sofort aktualisiert werden können. RESTful API ist verfügbar für die Anpassung oder das Hinzufügen von Motor-Features. Ein oVirt-Knoten ist ein Server mit RHEL, CentOS, Scientific Linux oder experimentell Debian, mit KVM Hypervisor aktiviert und ein VDSM (Virtual Desktop und Server Manager) Daemon geschrieben in Python. Das Management von Ressourcen, die von einem webadmin Portal initiiert werden, wird durch den Motor-Backend gesendet, der entsprechende Anrufe an den VDSM-Daemon ausgibt. VDSM steuert alle Ressourcen, die dem Knoten (Compute, Storage, Networking) und virtuellen Maschinen zur Verfügung stehen und ist auch für die Rückmeldung an den Motor über alle initiierten Vorgänge verantwortlich. Mehrere Knoten können aus dem oVirt Motor webadmin Portal geclustert werden, um RAS zu verbessern. Die oVirt-Engine kann auf einem Standalone-Server installiert oder auf einem Cluster von Knoten selbst innerhalb einer virtuellen Maschine (Selbst-Hosted-Engine) gehostet werden. Der selbstgehostete Motor kann manuell installiert oder automatisch über ein virtuelles Gerät bereitgestellt werden.o Virt wird auf mehreren anderen Projekten wie libvirt, Gluster, PatternFly und Ansible gebaut. Features Virtuelle Rechenzentren, verwaltet von oVirt, werden in Speicher, Netzwerk und Cluster, die aus einem oder mehreren oVirt-Knoten bestehen kategorisiert. Die Datenintegrität wird durch Fencing gewährleistet, mit Agenten, die verschiedene Ressourcen wie Basisboard-Management-Controller oder unterbrechungsfreie Stromversorgungen verwenden können. Die Speicherung wird innerhalb von Einrichtungen, die Speicherdomänen genannt werden, organisiert und kann lokal oder geteilt werden. Speicherdomänen können mit folgenden Speicherlösungen oder Protokollen erstellt werden: NFS iSCSI Fibre Channel Das POSIX konforme Dateisystem GlusterFSNetwork Management ermöglicht es, mehrere VLANs zu definieren, die an die auf den Knoten verfügbaren Netzwerkschnittstellen überbrückt werden können. Die Konfiguration von gebundenen Schnittstellen, IP-Adressen, Subnetmasken und Gateways auf verwalteten Knoten werden innerhalb der webadmin Portalschnittstelle unterstützt, ebenso wie SR-IOV auf Hardwarekonfigurationen, die diese Funktion unterstützen. Management-Funktionen für Rechenressourcen umfassen CPU-Pinning, definieren NUMA-Topologie, ermöglichen Kernel gleichseitiges Merging, Speicherüberprovisioning, HA VM-Reservierung etc. Das virtuelle Maschinenmanagement ermöglicht die Auswahl von hoher Verfügbarkeitspriorität, Live-Migration, Live-Snapshots, das Klonen virtueller Maschinen aus Snapshots, das Erstellen virtueller Maschinenvorlagen, die Verwendung von Cloud-Init zur automatisierten Konfiguration bei der Bereitstellung und Bereitstellung virtueller Maschinen. Zu den unterstützten Guest-Betriebssystemen gehören Linux, Microsoft Windows und FreeBSD. Zugriff auf virtuelle Maschinen kann von webadmin Portal mit SPICE, VNC und RDP Protokollen erreicht werden.o Virt kann mit vielen Open Source-Projekten, einschließlich OpenStack Glance und Neutron für Festplatten- und Netzwerkbereitstellung, Foreman/Katello für VM/Node Bereitstellung oder Ziehen relevanter Errata-Informationen in webadmin Portal integriert werden und kann mit ManageIQ für ein komplettes virtuelles Infrastruktur-Lebenszyklusmanagement weiter integriert werden. Desaster Recovery-Funktionen umfassen die Möglichkeit, jede Speicher-Domain in verschiedene oVirt-Instanzen zu importieren und Replikation kann von oVirt mit GlusterFS-Geo-Replikationsfunktion verwaltet werden, oder durch die Verwendung synchroner/asynchroner Block-Level-Replikation von Speicherhardware-Anbietern.o Virt-Engine-Backups können automatisiert und periodisch auf einen Remote-Standort übertragen werden.o Virt unterstützt hyperkonvergente Infrastruktur-Bereitstellungsszenarien.Self-hosted-Engine und Gluster-basierte Speicherdomänen ermöglichen eine zentrale Verwaltung aller Ressourcen, die nahtlos erweitert werden können, einfach durch Hinzufügen einer entsprechenden Anzahl von Knoten in den Cluster, ohne einzelne Punkte von fail.o Virt bietet eine tiefe Integration mit Gluster, einschließlich Gluster spezifische Leistungsverbesserungen.
