Eine Website (auch als Website geschrieben) ist eine Sammlung von Webseiten und verwandten Inhalten, die durch einen gemeinsamen Domainnamen identifiziert und auf mindestens einem Webserver veröffentlicht wird. Bemerkenswerte Beispiele sind wikipedia.org, google.com und amazon.com. Alle öffentlich zugänglichen Webseiten bilden das World Wide Web. Es gibt auch private Websites, die nur auf ein privates Netzwerk zugreifen können, wie die interne Website eines Unternehmens für seine Mitarbeiter. Websites sind in der Regel einem bestimmten Thema oder Zweck gewidmet, wie Nachrichten, Bildung, Handel, Unterhaltung oder soziale Netzwerke. Hyperlinking zwischen Webseiten führt die Navigation der Seite, die oft mit einer Startseite beginnt. Benutzer können auf Websites auf eine Reihe von Geräten zugreifen, darunter Desktops, Laptops, Tablets und Smartphones. Die auf diesen Geräten verwendete App wird als Webbrowser bezeichnet. Statische Website Eine statische Website ist eine, die Webseiten auf dem Server im Format gespeichert hat, das an einen Client-Webbrowser gesendet wird. Es wird in erster Linie in Hypertext Markup Language (HTML;) Cascading Style Sheets (CSS) verwendet, um das Aussehen über grundlegende HTML zu steuern. Bilder werden häufig verwendet, um das gewünschte Aussehen und als Teil der Hauptinhalte zu erstellen. Audio oder Video kann auch als statischer Inhalt betrachtet werden, wenn er automatisch spielt oder im Allgemeinen nicht interaktiv ist. Diese Art von Website zeigt in der Regel die gleichen Informationen für alle Besucher. Ähnlich wie bei der Aushändigung einer gedruckten Broschüre an Kunden oder Kunden wird eine statische Website in der Regel einheitliche, Standardinformationen für einen längeren Zeitraum bereitstellen. Obwohl der Website-Besitzer regelmäßig Updates vornehmen kann, ist es ein manueller Prozess, um den Text, Fotos und andere Inhalte zu bearbeiten und kann grundlegende Website-Design-Fähigkeiten und Software erfordern. Einfache Formulare oder Marketingbeispiele von Websites, wie klassische Website, eine fünfseitige Website oder eine Broschüre-Website sind oft statische Webseiten, weil sie vordefinierte statische Informationen an den Benutzer darstellen. Dies kann Informationen über ein Unternehmen und seine Produkte und Dienstleistungen durch Text, Fotos, Animationen, Audio/Video und Navigationsmenüs enthalten. Statische Webseiten können weiterhin Serverseite (SSI) als Bearbeitungskomfort verwenden, wie z.B. eine gemeinsame Menüleiste auf vielen Seiten teilen. Da das Verhalten der Website zum Leser noch statisch ist, wird dies nicht als dynamische Website betrachtet. Dynamische Website Eine dynamische Website ist eine, die sich häufig und automatisch verändert oder angepasst. Serverseitige dynamische Seiten werden durch den Computercode "on the fly" erzeugt, der das HTML erzeugt (CSS ist für das Aussehen verantwortlich und sind somit statische Dateien.) Es gibt eine breite Palette von Software-Systemen, wie CGI, Java Servlets und Java Server Pages (JSP,) Active Server Pages und ColdFusion (CFML), die zur Erzeugung dynamischer Web-Systeme und dynamischer Websites zur Verfügung stehen. Verschiedene Web Application Frameworks und Web-Template-Systeme sind für allgemeine Programmiersprachen wie Perl, PHP, Python und Ruby zur schnelleren und einfacheren Erstellung komplexer dynamischer Webseiten verfügbar. Eine Website kann den aktuellen Zustand eines Dialogs zwischen den Benutzern anzeigen, eine sich ändernde Situation überwachen oder Informationen in gewisser Weise personalisiert auf die Anforderungen des einzelnen Benutzers. Wenn zum Beispiel die Titelseite einer Nachrichtenseite angefordert wird, kann der auf dem Webserver laufende Code gespeicherte HTML-Fragmente mit Nachrichtenstorys kombinieren, die von einer Datenbank oder einer anderen Website über RSS abgerufen werden, um eine Seite zu erstellen, die die neuesten Informationen enthält. Dynamische Websites können interaktiv sein, indem Sie HTML-Formulare verwenden, Browser-Cookies speichern und lesen oder eine Reihe von Seiten erstellen, die die vorherige Geschichte der Klicks widerspiegeln. Ein weiteres Beispiel für dynamische Inhalte ist, wenn eine Einzelhandelswebsite mit einer Datenbank von Medienprodukten es einem Benutzer ermöglicht, eine Suchanfrage einzugeben, z.B. für das Keyword Beatles. Als Antwort wird der Inhalt der Webseite spontan die Art und Weise ändern, wie sie zuvor aussah, und dann eine Liste von Beatles-Produkten wie CDs, DVDs und Büchern anzeigen. Dynamische HTML verwendet JavaScript-Code, um den Webbrowser zu instruktieren, wie man die Seiteninhalte interaktiv ändert. Eine Möglichkeit, eine bestimmte Art von dynamischer Website zu simulieren und gleichzeitig den Leistungsverlust der Einleitung des dynamischen Motors auf einer Per-User- oder Per-Connections-Basis zu vermeiden, besteht darin, eine große Reihe von statischen Seiten periodisch automatisch zu regenerieren.
