Ein Uniform Resource Locator (URL) kolloquial als Webadresse bezeichnet, ist ein Verweis auf eine Web-Ressource, die seinen Standort auf einem Computernetzwerk und einen Mechanismus zum Abruf spezifiziert. Eine URL ist eine bestimmte Art von Uniform Resource Identifier (URI), obwohl viele Menschen die beiden Begriffe austauschbar verwenden. URLs treten am häufigsten auf Referenz-Webseiten (http) auf, werden aber auch für Dateiübertragung (ftp,) E-Mail (mailto,) Datenbankzugriff (JDBC) und viele andere Anwendungen verwendet. Die meisten Webbrowser zeigen die URL einer Seite über der Seite in einer Adressleiste. Eine Pfadkomponente, bestehend aus einer Folge von durch einen Slash )(/ getrennten Pfadsegmenten. Für ein URI ist immer ein Pfad definiert, der definierte Pfad kann jedoch leer sein (Nulllänge). Ein Segment kann auch leer sein, was zu zwei aufeinanderfolgenden Slashes (/) in der Bahnkomponente führt. Eine Pfadkomponente kann genau einem Dateisystempfad ähnlich oder abbilden, bedeutet aber nicht immer eine Beziehung zu einem. Wenn eine Autoritätskomponente fehlt, kann der Pfad nicht mit einem leeren Segment beginnen, also mit zwei Slashes (/), da die folgenden Zeichen als Autoritätskomponente interpretiert werden. Internationalisierte URL-Internet-Benutzer werden weltweit mit einer Vielzahl von Sprachen und Alphabeten verteilt und erwarten, dass sie URLs in ihren eigenen lokalen Alphabeten erstellen können. Ein Internationalized Resource Identifier (IRI) ist eine Form der URL, die Unicode-Zeichen enthält. Alle modernen Browser unterstützen IRIs. Die Teile der URL, die eine spezielle Behandlung für verschiedene Alphabete erfordern, sind der Domänenname und Pfad. Der Domainname im IRI ist als Internationalized Domain Name (IDN) bekannt.
