Ein Home-Server ist ein Computer-Server in einem privaten Computer-Residenz, der Dienstleistungen für andere Geräte innerhalb oder außerhalb des Haushalts über ein Heimnetz oder das Internet bietet. Solche Dienste können Datei- und Druckerdienste, Medienzentrum dienen, Home Automation Control, Web-Service (auf dem Netzwerk oder Internet,) Web-Caching, Datei-Sharing und Synchronisation, Videoüberwachung und digitale Videorekorder, Kalender und Kontakt-Sharing und Synchronisation, Konto-Authentifizierung und Backup-Dienste. Aufgrund der relativ geringen Anzahl von Computern auf einem typischen Heimnetz benötigt ein Home-Server häufig keine signifikante Rechenleistung. Home-Server können do-it-yourself-Stil mit einem wiederverwendeten, älteren Computer oder einem Plug-Computer implementiert werden; vorkonfigurierte kommerzielle Home-Server-Geräte sind ebenfalls verfügbar. Eine unterbrechungsfreie Stromversorgung wird manchmal bei Stromausfällen verwendet, die möglicherweise Daten verfälschen können. Die von Home-Servern bereitgestellten Dienste Administration und Konfiguration Home-Server laufen oft kopflos und können über eine Befehlsschale oder grafisch über ein Remote-Desktopsystem wie RDP, VNC, Webmin, Apple Remote Desktop oder viele andere verwaltet werden. Einige Home-Server-Betriebssysteme (z.B. Windows Home Server) umfassen eine verbraucherorientierte grafische Benutzeroberfläche (GUI) zur Einrichtung und Konfiguration, die auf Heimcomputern im Heimnetz (und remote über das Internet über Remote-Zugriff) zur Verfügung steht. Andere ermöglichen es Benutzern, native Betriebssystem-Tools für die Konfiguration zu verwenden. Zentrale Lagerung Home-Server fungieren oft als Netzwerkspeicher (NAS), der den Hauptvorteil bietet, dass alle Benutzerdateien zentral und sicher gespeichert werden können, mit flexiblen Berechtigungen, die auf sie angewendet werden. Solche Dateien können leicht von jedem anderen System im Netzwerk aufgerufen werden, sofern die korrekten Anmeldeinformationen geliefert werden. Dies gilt auch für geteilte Drucker. Solche Dateien können auch über das Internet geteilt werden, um von überall auf der Welt mit Fernzugriff zugänglich zu sein. Server mit Unix oder Linux mit der kostenlosen Samba-Suite (oder bestimmte Windows Server-Produkte - Windows Home Server ausgenommen) können Domänensteuerung, benutzerdefinierte Logon-Skripte und Roaming-Profile für Benutzer bestimmter Versionen von Windows zur Verfügung stellen. Dies ermöglicht es einem Benutzer, sich von jeder Maschine in der Domain anzumelden und Zugriff auf sie oder seine "Meine Dokumente" und personalisierte Windows- und Anwendungspräferenzen zu haben - mehrere Konten auf jedem Computer im Haus sind nicht erforderlich. Medien Home-Server werden oft verwendet, um Multimedia-Inhalte zu bedienen, einschließlich Fotos, Musik und Video auf andere Geräte im Haushalt (und sogar im Internet; siehe Raumverschiebung, Tonido und Orb). Mit Standard-Protokollen wie DLNA oder proprietären Systemen wie iTunes, können Benutzer ihre auf dem Home-Server gespeicherten Medien von jedem Zimmer im Haus zugreifen. Windows XP Media Center Edition, Windows Vista und Windows 7 können als Home-Server fungieren und eine bestimmte Art von Medien unterstützen, die die interaktive Benutzererfahrung an Media Center Extenders einschließlich der Xbox 360 streamt. Windows Home Server unterstützt Medienstreaming auf Xbox 360 und andere DLNA-basierte Medienempfänger über die integrierte Windows Media Connect Technologie. Einige Windows Home Server Gerätehersteller, wie HP, erweitern diese Funktionalität mit einer vollen DLNA Implementierung wie PacketVideo TwonkyMedia Server. Es gibt viele Open-Source- und vollfunktionelle Programme für Medien, die für Linux zur Verfügung stehen. LinuxMCE ist ein Beispiel, das es anderen Geräten erlaubt, ein Festplattenbild auf dem Server abzu booten und so Geräte wie Set-Top-Boxen zu werden. Asterisk, Xine, MythTV (eine weitere Medienlösung), VideoLAN, SlimServer, DLNA und viele andere Open-Source-Projekte sind vollständig für ein nahtloses Heimkino/Automation/Telefonie-Erlebnis integriert. Auf einem Apple Macintosh Server, Optionen umfassen iTunes, PS3 Media Server und Elgato. Zusätzlich kann Boxee für Macs, die direkt mit TVs verbunden sind, als vollwertiges Media Center-Interface fungieren. Server sind in der Regel immer auf, so dass die Zugabe eines TV- oder Radiotuners die Aufnahme jederzeit möglich macht. Einige Home-Server bieten Remote-Zugriff auf Medien und Entertainment-Inhalte. Fernzugriff Ein Home-Server kann verwendet werden, um Remote-Zugriff in das Haus von Geräten im Internet, mit Remote-Desktop-Software und anderen Remote-Administration-Software. Zum Beispiel bietet Windows Home Server einen Remote-Zugriff auf Dateien, die auf dem Home-Server über eine Web-Schnittstelle gespeichert werden, sowie einen Remote-Zugriff auf Remote Desktop-Sitzungen auf PCs im Haus.Ebenso bietet Tonido direkten Zugriff über einen Webbrowser aus dem Internet, ohne dass eine Portweiterleitung oder andere Einrichtung erforderlich ist. Einige Enthusiasten nutzen oft VPN-Technologien. Auf einem Linux-Server sind zwei beliebte Tools (unter vielen) VNC und Webmin. VNC ermöglicht es Kunden, einen Server-GUI-Desktop remote anzuzeigen, als ob der Benutzer physisch vor dem Server sitzt. Eine GUI muss dafür nicht auf der Server-Konsole ausgeführt werden; es können mehrere virtuelle Desktop-Umgebungen gleichzeitig geöffnet werden. Webmin ermöglicht es Benutzern, viele Aspekte der Serverkonfiguration und Wartung von einer einfachen Web-Schnittstelle zu steuern. Beide können konfiguriert werden, um von überall im Internet zuzugreifen. Server können auch über die Kommandozeilen-basierten Telnet- und SSH-Protokolle remote aufgerufen werden. Einige Benutzer wählen, um einen Webserver zu betreiben, um Dateien einfach und öffentlich (oder privat, im Heimnetz) zu teilen. Andere stellen Web-Seiten ein und dienen ihnen direkt von ihrem Zuhause, obwohl dies kann gegen einige ISPs Nutzungsbedingungen sein. Manchmal werden diese Webserver auf einem nicht standardmäßigen Port ausgeführt, um die Portsperre des ISP zu vermeiden. Beispiele für Webserver, die auf Heimservern verwendet werden, sind Apache und IIS. Viele andere Webserver sind verfügbar; siehe Vergleich der leichten Webserver, Vergleich der Webserver. Web-Proxy Einige Netzwerke haben eine HTTP-Proxy, die verwendet werden kann, um den Web-Zugriff zu beschleunigen, wenn mehrere Benutzer die gleichen Websites besuchen, und um vorbei zu blockieren Software, während der Besitzer das Netzwerk einer Institution verwendet, die bestimmte Websites blockieren könnte. Öffentliche Proxys sind oft langsam und unzuverlässig und so lohnt es sich, die eigene private Proxy einzurichten. Einige Proxies können konfiguriert werden, um Webseiten auf dem lokalen Netzwerk zu blockieren, wenn es als transparente Proxy eingerichtet ist. E-Mail Viele Home-Server führen auch E-Mail-Server, die E-Mail für den Domainnamen des Eigentümers behandeln. Die Vorteile sind viel größere Postfächer und maximale Nachrichtengröße als die meisten kommerziellen E-Mail-Dienste. Zugriff auf den Server, da es auf dem lokalen Netzwerk ist viel schneller als mit einem externen Dienst. Dies erhöht auch die Sicherheit, da E-Mails nicht auf einem Off-Site-Server wohnen. Gopher Ein ungewöhnlicher Service ist das Gopher-Protokoll, ein Hypertext-Dokument-Retrieval-Protokoll, das das World Wide Web vorgetäuscht und in den frühen 1990er Jahren populär war. Viele der verbleibenden Gopher-Server werden von Home-Servern unter Verwendung von PyGopherd und dem Bucktooth Gopher-Server ausgeführt.
