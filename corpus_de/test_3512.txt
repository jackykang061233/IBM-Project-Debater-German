IP-Routing ist das Gebiet der Routing-Methoden von Internet Protocol (IP) Paketen innerhalb und über IP-Netzwerke. Dazu gehören nicht nur Protokolle und Technologien, sondern auch die Politik der weltweiten Organisation und Konfiguration der Internetinfrastruktur. In jedem IP-Netzwerkknoten umfasst IP-Routing die Bestimmung eines geeigneten Pfads für ein Netzwerkpaket von einer Quelle zu seinem Ziel in einem IP-Netzwerk. Das Verfahren verwendet statische Konfigurationsregeln oder dynamisch gewonnene Zustandsinformationen, um bestimmte Paketweiterleitungsmethoden auszuwählen, um den Traffic auf den nächsten verfügbaren Zwischennetzknoten zu lenken, der dem gewünschten Endziel näher kommt, wobei ein Gesamtpfad, der mehrere Computernetzwerke überspannt. Netzwerke werden durch spezialisierte Hosts, sogenannte Gateways oder Router mit spezialisierter Softwareunterstützung, die für das Routing optimiert ist, voneinander getrennt. Bei Routern werden Pakete, die an einer Schnittstelle ankommen, auf Quell- und Zieladressen untersucht und an die entsprechende ausgehende Schnittstelle entsprechend ihrer Zieladresse und einer Reihe von Regeln und Kennzahlen abgefragt. Regeln werden in einer Routing-Tabelle codiert, die Einträge für alle Schnittstellen und deren angeschlossene Netzwerke enthält. Wenn keine Regel die Anforderungen an ein Netzwerkpaket erfüllt, wird diese an eine Standardroute weitergeleitet. Routing-Tabellen werden entweder manuell von einem Netzwerkadministrator gehalten oder dynamisch mit einem Routing-Protokoll aktualisiert. Routing-Regeln können andere Parameter enthalten als Quelle und Ziel, wie Einschränkungen auf verfügbare Bandbreite, erwartete Paketverlustraten und spezifische Technologieanforderungen. IP-Forwarding-Algorithmen berücksichtigen die Größe jedes Pakets, die Art des im Header angegebenen Dienstes sowie die Eigenschaften der verfügbaren Links zu anderen Routern im Netzwerk, wie Link-Kapazität, Auslastung und maximale Datengrammgröße, die auf dem Link unterstützt wird. Im Allgemeinen bestimmt die meisten Routing-Software eine Route durch einen kürzesten Pfad-Algorithmus. Andere Routing-Protokolle können jedoch andere Metriken zur Bestimmung des besten Pfades verwenden. Aufgrund der für jeden Link erforderlichen und vorhandenen Metriken hat jeder Pfad einen entsprechenden Aufwand. Der Routing-Algorithmus versucht, die Kosten bei der Auswahl des nächsten Hopfens zu minimieren. Ein Routing-Protokoll ist ein Software-Mechanismus, mit dem Router Informationen über die Topologie des Netzwerks und die Fähigkeiten jedes Routing-Knotens kommunizieren und teilen. Sie implementiert somit die netzglobalen Regeln, mit denen der Verkehr innerhalb eines Netzes und über mehrere Netzwerke geleitet wird. Verschiedene Protokolle werden häufig für unterschiedliche Topologien oder verschiedene Anwendungsbereiche verwendet. Zum Beispiel wird das Open Shortest Path First (OSPF) Protokoll in der Regel zum Routing von Paketen zwischen Subnetzen innerhalb eines Unternehmens und dem Border Gateway Protocol (BGP) weltweit verwendet. BGP ist der de facto Standard der weltweiten Internet-Routing. Protokollklassifikation Routing-Protokolle können im Hinblick auf den Netzumfang durch ihren Einsatzbereich weitgehend unterschieden werden. Interne Gateway-Protokolle werden für das Routing innerhalb autonomer Systeme verwendet, während externe Gateway-Protokolle den Verkehr zwischen ihnen führen. Die ehemalige Gruppe wird durch das Routing Information Protocol (RIP) und Open Shortest Path First (OSPF) veranschaulicht, während das Exterior Gateway Protocol (EGP) und das Border Gateway Protocol (BGP) Beispiele für den äußeren Typ sind. BGP ist das im Internet verwendete dominante Routenverteilungsprotokoll. Routing-Algorithmus Der IP-Forwarding-Algorithmus ist eine spezifische Implementierung von Routing für IP-Netzwerke. Um eine erfolgreiche Übertragung von Daten zu erreichen, verwendet der Algorithmus eine Routing-Tabelle, um als nächstes Ziel für ein Datengramm einen nächsten Router auszuwählen. Die IP-Adresse des ausgewählten Routers ist als Next-Hop-Adresse bekannt. Wenn mehrere Ziele übereinstimmen, wird die Route mit der längsten Subnetmaske gewählt (die spezifischeste). Wenn es mehrere Routen mit derselben Subnetmaske gibt, wird die Route mit der niedrigsten Metrik verwendet. Wenn es mehrere Standardrouten gibt, wird die Metrik auch verwendet, um zu bestimmen, welche verwendet werden soll. Wenn es mehrere Routen mit der gleichen Subnetmaske und Metrik gibt, kann das System als Weiterleitungsstrategie gleiche Kosten-Multi-Pfad-Routing verwenden. Der IP-Forwarding-Algorithmus sagt: Bei einer Ziel-IP-Adresse, D und Netzwerk-Präfix, N:if (N entspricht einer direkt angeschlossenen Netzwerkadresse ) Liefern Sie Datagram an D über diese Netzwerkverbindung; auch wenn ( Die Routing-Tabelle enthält eine Route für N ) Senden Sie Datagram an die in der Routing-Tabelle aufgeführte nächste Hop-Adresse; sonst, wenn (eine Standardroute existiert) Senden Sie Datagram an die Standardroute; sonst Senden Sie eine weiterleitende Fehlermeldung an den Originator; Wenn keine Route verfügbar ist, wird eine ICMP-Fehlermeldung an den Originator des Pakets gesendet, um zu informieren, dass Host das Paket nicht geliefert werden konnte, und um unnötige Retransmission zu vermeiden Netzwerk-Verstopfung. Der sendende Host sollte entweder die Übertragung stoppen oder eine andere Adresse oder Route wählen. Der Host verfügt über mehrere Netzwerkschnittstellen.eth0 ist der Schnittstellenname der Netzwerkschnittstellenkarte, die einen Ethernet-Port repräsentiert.ppp0 ist eine PPPoE-Schnittstelle, die in diesem Beispiel als Standardroute konfiguriert ist. Eine Standardroute wird durch das Ziel 0.0.0.0 erkannt und die Flagge G. Ein Netzwerkrouter wird durch die Netzwerkmaske 255.255.255.255.255 und die Flagge H identifiziert.
