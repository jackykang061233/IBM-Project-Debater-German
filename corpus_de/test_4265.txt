Komprimierung Verschiedene Systeme können verwendet werden, um die Größe der zu speichernden Quelldaten so zu schrumpfen, dass sie weniger Speicherplatz verwendet. Komprimierung ist häufig ein integriertes Feature von Bandantrieb Hardware. Mängel Redundanz durch ein Backup ähnlich konfigurierter Workstations kann reduziert werden, so dass nur eine Kopie gespeichert wird. Diese Technik kann auf Datei- oder Rohblockebene angewendet werden. Diese potenziell große Reduktion wird als Deduplizierung bezeichnet. Es kann auf einem Server auftreten, bevor sich Daten zu Backup-Medien bewegen, manchmal als Source/Client-Seitendeduplikation bezeichnet. Dieser Ansatz reduziert auch die Bandbreite, die benötigt wird, um Backup-Daten an seine Zielmedien zu senden. Der Vorgang kann auch an der Zielspeichereinrichtung erfolgen, manchmal auch als Inline oder Backend-Deduplikation bezeichnet. Vervielfältigung Manchmal werden Backups auf einen zweiten Satz von Speichermedien dupliziert. Dies kann getan werden, um die Archivdateien zu ändern, um die Wiederherstellungsgeschwindigkeit zu optimieren, oder um eine zweite Kopie an einem anderen Ort oder auf einem anderen Speichermedium zu haben – wie in der Disk-to-disk-to-Tape-Fähigkeit der Enterprise Client-Server-Backup. Verschlüsselung hochkarätig abnehmbare Speichermedien wie Backup-Bands stellen ein Datensicherheitsrisiko dar, wenn sie verloren oder gestohlen werden. Die Verschlüsselung der Daten auf diesen Medien kann dieses Problem mildern, jedoch ist die Verschlüsselung ein CPU-intensiver Prozess, der Backup-Geschwindigkeiten verlangsamen kann, und die Sicherheit der verschlüsselten Backups ist nur so effektiv wie die Sicherheit der Schlüsselverwaltungspolitik.Multiplexing Wenn es viele weitere Computer gibt, die gesichert werden sollen, als es Zielspeicher gibt, kann die Möglichkeit, ein einziges Speichergerät mit mehreren gleichzeitigen Sicherungen zu verwenden, nützlich sein. Allerdings wird das geplante Backup-Fenster über "multiplexed backup" nur für Bandziele verwendet. Herstellung Der Prozess der Neuanordnung der Backups in einer Archivdatei ist als Refactoring bekannt. Zum Beispiel, wenn ein Backup-System verwendet ein einzelnes Band jeden Tag, um die inkrementalen Backups für alle geschützten Computer zu speichern, Wiederherstellung eines der Computer könnte viele Bänder erfordern. Refactoring könnte verwendet werden, um alle Backups für einen einzigen Computer auf einem einzigen Band zu konsolidieren, wodurch eine "synthetische vollständige Sicherung" entsteht. Dies ist besonders nützlich für Backup-Systeme, die Inkrementale für immer Stil Backups tun. Staging Manchmal werden Backups auf eine Staging-Disk kopiert, bevor sie auf Band kopiert werden. Dieser Prozess wird manchmal als D2D2T bezeichnet, ein Akronym für Disk-to-disk-to-Tape. Es kann nützlich sein, wenn es ein Problem gibt, das der Geschwindigkeit des Endzielgerätes mit dem Quellgerät entspricht, wie es häufig in netzwerkbasierten Backup-Systemen vorkommt. Es kann auch als zentralisierter Ort für die Anwendung anderer Datenmanipulationstechniken dienen. Zielvorgaben zur Wiedereinziehung (RPO) : Der Zeitpunkt, an dem sich die neu gestartete Infrastruktur widerspiegelt, wird als "die maximale Zieldauer, in der Daten (Transaktionen) aufgrund eines großen Vorfalls von einem IT-Service verloren gehen könnten" ausgedrückt. Im Wesentlichen ist dies die Rückrolle, die durch die Rückgewinnung erfahren wird. Der wünschenswerteste RPO wäre der Punkt kurz vor dem Datenverlust-Ereignis. Ein neuerer Wiederherstellungspunkt erreichbar erfordert eine Erhöhung der Frequenz der Synchronisation zwischen den Quelldaten und dem Backup-Repository. Rückgewinnungszeitziel (RTO): Die Zeit zwischen Katastrophe und Wiederherstellung von Geschäftsfunktionen verstrichen. Datensicherheit : Neben der Erhaltung des Zugriffs auf Daten für seine Eigentümer müssen Daten von unberechtigtem Zugriff eingeschränkt werden. Backups müssen in einer Weise durchgeführt werden, die das Unternehmen des ursprünglichen Eigentümers nicht beeinträchtigt. Dies kann mit der Datenverschlüsselung und den richtigen Medienhandling-Richtlinien erreicht werden. Aufbewahrungsfrist : Verordnungen und Richtlinien können zu Situationen führen, in denen Backups für einen bestimmten Zeitraum erwartet werden, aber nicht weiter. Die Sicherungen nach dieser Zeit können zu unerwünschter Haftung und suboptimaler Nutzung von Speichermedien führen. Kontrollsumme oder Hash-Funktionsvalidierung : Anwendungen, die bis zu Tape-Archivdateien sichern, benötigen diese Option, um zu überprüfen, ob die Daten genau kopiert wurden. Überwachung des Sicherungsvorgangs : Enterprise-Client-Server-Backup-Anwendungen benötigen eine Benutzeroberfläche, die es Administratoren ermöglicht, den Backup-Prozess zu überwachen, und beweist die Einhaltung von Regulierungsbehörden außerhalb der Organisation; zum Beispiel könnte ein Versicherungsunternehmen in den USA unter HIPAA erforderlich sein, um zu zeigen, dass seine Client-Daten die Anforderungen an die Speicherung von Datensätzen erfüllen. Benutzerinitiierte Backups und Restaurationen : Um kleinere Katastrophen zu vermeiden oder wiederherzustellen, wie z.B. unbeabsichtigt das Löschen oder Überschreiben der guten Versionen einer oder mehrerer Dateien, kann der Computerbenutzer – anstatt ein Administrator – Backups und Restaurationen (aus nicht notwendigerweise der hochrangigsten Sicherung) von Dateien oder Ordnern initiieren.
