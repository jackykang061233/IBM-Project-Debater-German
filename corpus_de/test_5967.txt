Vor einem verletzlichen System ist der Angriff rechnerisch kostengünstig und erfordert oft nur bekannten Ciphertext. Der Angriff kann als Signalerfassungsproblem behandelt werden. Das Signal besteht aus der zeitlichen Variation durch das Ziel-Exponent-Bit, und Rauschergebnisse aus Messungenauigkeiten und Timing-Variationen durch unbekannte Exponent-Bits. Die Eigenschaften von Signal und Rauschen bestimmen die Anzahl der für den Angriff erforderlichen Zeitmessungen. Timing-Angriffe können möglicherweise gegen andere Kryptosysteme verwendet werden, einschließlich symmetrischer Funktionen. Vorrechte Eskalation Eine einfache und generische Prozessor Backdoor kann von Angreifern als Mittel verwendet werden, um Privilegien zu erhalten, die denen eines bestimmten laufenden Betriebssystems gleichwertig sind. Auch kann ein nicht privilegierter Prozess einer der nicht privilegierten geladenen Domäne, die auf einem virtuellen Maschinenmonitor läuft, zu Privilegien gelangen, die denen des virtuellen Maschinenmonitors gleichwertig sind. Loïc Duflot studierte Intel-Prozessoren in der Zeitung "CPU-Bugs, CPU-Backdoors und Folgen auf Sicherheit" ; er erklärt, dass der Prozessor definiert vier verschiedene Privilegringe nummeriert von 0 (meist privilegiert) bis 3 (bevorzugt). Kernel-Code läuft in der Regel in Ring 0, während Benutzer-Raum-Code in der Regel in Ring läuft 3. Die Verwendung einiger sicherheitskritischer Montagesprachenanweisungen ist auf Ring 0-Code beschränkt. Um das Privileg durch die Hintertür zu eskalieren, muss der Angreifer: die Hintertür aktivieren, indem die CPU in den gewünschten Zustand gesteckt wird; Code injizieren und in Ring 0 ausführen; zurück zum Ring 3, um das System in einen stabilen Zustand zurückzugeben. Wenn der Code in Ring 0 läuft, funktionieren die Systemanrufe nicht: Das System in Ring 0 zu verlassen und einen zufälligen System Call-Ausgang zu betreiben) wird wahrscheinlich das System abstürzen. Die Backdoors Loïc Duflot Geschenke sind einfach, da sie nur das Verhalten von drei Montagesprachenanweisungen verändern und sehr einfache und spezifische Aktivierungsbedingungen haben, so dass sie sehr unwahrscheinlich aktiviert werden.
