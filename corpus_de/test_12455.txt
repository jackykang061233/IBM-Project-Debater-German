Android Oreo (codenamed Android O während der Entwicklung) ist die achte große Version und die 15. Version des Android mobilen Betriebssystems. Es wurde erstmals im März 2017 als Alpha-Qualitäts-Entwickler-Vorschau veröffentlicht und am 21. August 2017 veröffentlicht. Es enthält eine Reihe von wichtigen Funktionen, einschließlich Benachrichtigung Gruppierung, Bild-in-Bild-Unterstützung für Video-, Leistungsverbesserungen und Batterie-Nutzungsoptimierung, und Unterstützung für Autofiller, Bluetooth 5, System-Level-Integration mit VoIP-Apps, breite Farbpaletten und Wi-Fi-Aware. Android Oreo enthält integrierte Unterstützung für Bild-in-Bild-Modi (unterstützt in der YouTube-App für YouTube Premium-Abonnenten). Die Settings App verfügt über ein neues Design, das in der Größe reduziert wurde, mit einem weißen Thema und einer tieferen Kategorisierung von verschiedenen Einstellungen, während seine Klingeltöne, Alarm- und Benachrichtigungs-Sound-Einstellungen jetzt eine Option zum Hinzufügen von benutzerdefinierten Sounds in die Liste enthalten. Das Android 8.1 Update unterstützt die Anzeige von Batterieprozentsätzen für angeschlossene Bluetooth-Geräte, macht den Benachrichtigungston leicht transluzent, und dimmt die on-screen Navigationstasten, um die Möglichkeit des Einbrennens zu reduzieren. Plattform Android Oreo fügt Unterstützung für Nachbarschafts-Aware Networking (NAN) für Wi-Fi basierend auf Wi-Fi-Aware, Bluetooth 5, breite Farb-Gamuts in Apps, eine API für Autofiller, Multiprozess und Google Browsing-Unterstützung für WebViews, eine API, um System-Level-Integration für VoIP-Apps zu ermöglichen, und Start-Aktivitäten auf Remote-Displays. Android Runtime (ART) bietet Leistungsverbesserungen. Android Oreo enthält zusätzliche Grenzwerte für die Hintergrundaktivitäten von Apps, um die Akkulaufzeit zu verbessern. Apps können "Adaptive Icons" für unterschiedlich geformte Behälter angeben, die durch Themen wie Kreise, Quadrate und Squircles vorgegeben werden. Android Oreo fügt native Unterstützung für Advanced Audio Coding, aptX, aptX HD und LDAC Bluetooth-Codecs hinzu. Android Oreo unterstützt neue Emoji, die im Unicode 10 Standard enthalten waren. Es wurde auch eine neue Emoji-Schrift eingeführt, die insbesondere seine Gesichtsfiguren neu gestaltet, um eine traditionelle kreisförmige Form zu verwenden, im Gegensatz zu dem auf KitKat eingeführten Blob-Design. Die zugrunde liegende Architektur von Android wurde überarbeitet, so dass Low-Level, herstellerspezifischer Code zur Unterstützung der Hardware eines Geräts durch eine Hardware-Abstraktionsschicht, die als "Anbieter-Schnittstelle" bekannt ist, vom Android OS-Rahmen getrennt werden kann. Vendor Interfaces müssen mit zukünftigen Versionen von Android kompatibel gemacht werden. Diese neue Architektur, genannt Project Treble, ermöglicht die schnellere Entwicklung und Bereitstellung von Android-Updates für Geräte, da Anbieter nur die notwendigen Änderungen an ihrer gebündelten Software vornehmen müssen.Alle Geräte, die mit Oreo verschiffen, müssen eine Herstellerschnittstelle unterstützen, diese Funktion ist jedoch optional für Geräte, die von einer früheren Version auf Oreo aktualisiert werden. Das in Android 7.0 eingeführte "Seamless-Updates"-System wurde auch geändert, um Update-Dateien direkt auf die Systempartition herunterzuladen, anstatt sie zuerst auf die Benutzerpartition herunterzuladen. Dies reduziert den Speicherplatzbedarf für Systemaktualisierungen. Android Oreo stellt ein neues automatisches Reparatursystem vor, das als "Rescue Party" bekannt ist; wenn das Betriebssystem erkennt, dass Kernsystemkomponenten während des Starts hartnäckig abstürzen, wird es automatisch eine Reihe von eskalierenden Reparaturschritten durchführen. Wenn alle automatischen Reparaturschritte erschöpft sind, wird das Gerät in den Wiederherstellungsmodus neu starten und bieten, um eine Werksrückstellung durchzuführen. Das Android 8.1 Update führt auch eine neurale Netzwerk-API ein, die auf ["provide]-Apps mit Hardwarebeschleunigung für on-device-Maschinen-Learning-Operationen entwickelt wird." Diese API ist für den Einsatz mit maschinellen Lernplattformen wie TensorFlow Lite und spezialisierten Co-Prozessoren wie dem Pixel Visual Core (featured in Googles Pixel 2 Smartphones, aber dormant bis 8.1 installiert ist), bietet aber auch einen CPU-Rückfallmodus.
