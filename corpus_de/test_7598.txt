Bei der Computer-Netzwerkierung ist ein dünner Client ein einfacher (low-performance) Computer, der für die Erstellung einer Remote-Verbindung mit einer serverbasierten Rechenumgebung optimiert wurde. Der Server macht die meisten Arbeiten, die das Starten von Softwareprogrammen, die Durchführung von Berechnungen und die Speicherung von Daten beinhalten können. Dies kontrastiert mit einem Fett-Client oder einem herkömmlichen Personal-Computer; der ehemalige ist auch für die Arbeit in einem Client-Server-Modell gedacht, hat jedoch eine erhebliche lokale Verarbeitungsleistung, während dieser seine Funktion vor allem lokal durchführen will. Dünne Clients treten als Komponenten einer breiteren Recheninfrastruktur auf, in der viele Clients ihre Berechnungen mit einem Server- oder Serverbetrieb teilen. Die serverseitige Infrastruktur nutzt Cloud-Computing-Software wie Application Virtualization, Hosted Shared Desktop (HSD) oder Desktop Virtualization (VDI). Diese Kombination bildet das sogenannte Cloud-basierte System, bei dem Desktop-Ressourcen in einem oder mehreren Rechenzentren zentralisiert werden. Die Vorteile der Zentralisierung sind Hardware-Ressourcenoptimierung, reduzierte Softwarewartung und verbesserte Sicherheit. Beispiel für Hardware-Ressourcen-Optimierung: Die Verkabelung, Bussing und I/O können minimiert werden, während Leerspeicher und Verarbeitungsleistung auf Benutzersitzungen angewendet werden können, die es am meisten benötigen. Beispiel für reduzierte Softwarewartung: Software-Patch- und Betriebssystem (OS)-Migrationen können für alle Benutzer in einem Fall angewendet, getestet und aktiviert werden, um das Roll-out zu beschleunigen und die Verwaltungseffizienz zu verbessern. Beispiel für eine verbesserte Sicherheit: Softwareanlagen werden zentralisiert und leicht feuerverstärkt, überwacht und geschützt. Sensitive Daten sind bei Desktop-Verlust oder Diebstahl nicht kompromisslos. Dünne Client-Hardware unterstützt in der Regel gemeinsame Peripheriegeräte wie Tastaturen, Mäuse, Monitore, Jacken für Sound Peripheriegeräte und offene Ports für USB-Geräte (z.B. Drucker, Flash-Laufwerk, Webcam). Einige dünne Clients umfassen serielle oder parallele Ports, um ältere Geräte zu unterstützen, z.B. Empfangsdrucker, Skalen oder Zeituhren. Dünne Client-Software besteht in der Regel aus einer grafischen Benutzeroberfläche (GUI,) Cloud Access Agents (z.B. RDP, ICA, PCoIP), einem lokalen Webbrowser, Terminal-Emulatoren (in einigen Fällen) und einem grundlegenden Satz von lokalen Diensten. Eigenschaften Architektur Bei der Verwendung von Cloud-basierter Architektur nimmt der Server die Bearbeitungsbelastung mehrerer Clientsitzungen auf, die als Host für jedes Endgerät fungieren. Die Client-Software ist eng zielgerichtet und leicht; daher muss nur der Host-Server oder Server-Bauernhof gesichert werden, anstatt auf jedem Endgerät installierte Software zu sichern (obwohl dünne Clients noch grundlegende Sicherheit und starke Authentifizierung benötigen, um unberechtigten Zugriff zu verhindern). Einer der kombinierten Vorteile der Verwendung von Cloud-Architektur mit dünnen Client-Desktops ist, dass kritische IT-Assets für eine bessere Nutzung von Ressourcen zentralisiert sind. Ungenutzter Speicher, Bussing-Lanes und Prozessor-Kerne innerhalb einer einzelnen Benutzersitzung können beispielsweise für andere aktive Benutzersitzungen genutzt werden. Die Einfachheit der dünnen Client-Hardware und Software führt zu sehr niedrigen Gesamtbetriebskosten, aber einige dieser anfänglichen Einsparungen können durch die Notwendigkeit einer robusteren Cloud-Infrastruktur auf der Serverseite ausgeglichen werden. Eine Alternative zur herkömmlichen Server-Bereitstellung, die die Infrastrukturkosten im Laufe der Zeit ausbreitet, ist ein Cloud-basiertes Abonnement-Modell, das als Desktop als Service bekannt ist, mit dem IT-Organisationen die Cloud-Infrastruktur an einen Dritten auslagern können. Ein Zero-Client nutzt sehr leichte Firmware, die lediglich die Netzwerkkommunikation über eine grundlegende GUI (Graphical User Interface) initialisiert und die vom Server empfangenen Informationen dekodiert und lokale Eingaben an den Host sendet. Ein Gerät mit einer solchen einfachen Funktionalität hat weniger Nachfrage nach komplexen Hardware oder Silizium und wird daher weniger anfällig für obsoleszenz. Ein weiterer wesentlicher Vorteil des Zero Client-Modells ist, dass seine leichte Firmware eine ultrakleine Angriffsfläche darstellt, die es sicherer macht als ein dünner Client.
