REF Schema (Resource Description Framework Schema, verschiedene abgekürzt als RDFS, RDF(S,) RDF-S oder RDF/S) ist eine Reihe von Klassen mit bestimmten Eigenschaften mit dem RDF-Extensible Knowledge Representations-Datenmodell, die grundlegende Elemente für die Beschreibung von Onlogien bietet. Es verwendet verschiedene Formen von RDF-Vokabularen, die dazu bestimmt sind, RDF-Ressourcen zu strukturieren. RDF und RDFS können in einer Dreifachhandlung gespeichert werden, dann kann man von ihnen einige Kenntnisse mit Hilfe einer Abfragesprache, wie SPARQL, mit sich bringen. Die erste Version wurde im April 1998 vom World-Wide Web Consortium (W3C) veröffentlicht und die letzte W3C Empfehlung wurde im Februar 2014 veröffentlicht. Viele RDFS-Komponenten sind in der ausdrucksvolleren Web Ontology Language (OWL) enthalten. Terminologie RDFS-Konstrukte sind die RDFS-Klassen, zugehörige Eigenschaften und Gebrauchseigenschaften, die auf dem Vokabular von RDF gebaut wurden. RDFS Ein Mitbestimmungsregime definiert nicht nur durch die Verwendung von RDFS (oder OWL, etc.), welche Mitbeziehungsbeziehung verwendet wird, sondern auch welche Abfragen und Graphen für das Regime gut ausgebildet sind. Die RDFS-Beziehung ist eine Standard-Beziehungsbeziehung in der semantischen Bahn. So erklärt z.B.: "Dog1 ist ein Tier", "Cat1 ist eine Katze", "Zoos-Hosttiere" und "Zoo1 beherbergt die Katze2": ex:dog1 rdf:type ex:animal ex:cat1 rdf: Typ ex:cat zoo:host rdfs:range ex:animal ex:zoo1 zoo:host ex:cat2 Der Graph ist nicht gut ausgebildet, weil das System nicht erraten kann, dass eine Katze ein Tier ist.
