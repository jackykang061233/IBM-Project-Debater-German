Microsoft Windows, allgemein als Windows bezeichnet, ist eine Gruppe von mehreren proprietären grafischen Betriebssystem-Familien, die alle von Microsoft entwickelt und vertrieben werden. Jede Familie kümmert sich um einen bestimmten Sektor der Computerindustrie. Active Microsoft Windows-Familien umfassen Windows NT und Windows IoT; diese können Unterfamilien umfassen (z.B. Windows Server oder Windows Embedded Compact) (Windows CE). Defunct Microsoft Windows-Familien umfassen Windows 9x, Windows Mobile und Windows Phone. Microsoft hat am 20. November 1985 eine Betriebsumgebung namens Windows eingeführt, als grafische Betriebssystemhülle für MS-DOS in Reaktion auf das wachsende Interesse an grafischen Benutzeroberflächen (GUIs). Microsoft Windows kam, um den Weltmarkt der persönlichen Computer (PC) Markt mit über 90% Marktanteil zu dominieren, überholt Mac OS, die 1984 eingeführt worden war. Apple kam, um Windows als unlautere Einmischung in ihre Innovation in GUI-Entwicklung zu sehen, wie sie auf Produkten wie die Lisa und Macintosh implementiert wurde (früher im Gericht in Microsofts Gunst 1993 niedergelegt). Windows ist auf PCs immer noch das beliebteste Betriebssystem in allen Ländern. Jedoch, im Jahr 2014, Microsoft gab zu verlieren die Mehrheit der gesamten Betriebssystem-Markt auf Android, wegen der massiven Wachstum im Vertrieb von Android-Smartphones. 2014 war die Anzahl der verkauften Windows-Geräte weniger als 25% der von Android-Geräten verkauft. Dieser Vergleich kann jedoch nicht vollständig relevant sein, da die beiden Betriebssysteme traditionell unterschiedliche Plattformen ansprechen. Dennoch zeigen Zahlen für die Servernutzung von Windows (die mit Wettbewerbern vergleichbar sind) einen dritten Marktanteil, ähnlich wie bei der Endbenutzernutzung. Windows 1.0 sollte mit Apples Betriebssystem konkurrieren, aber nur wenig Popularität erreicht. Windows 1.0 ist kein komplettes Betriebssystem, sondern erweitert MS-DOS. Die Shell von Windows 1.0 ist ein Programm, das als MS-DOS Executive bekannt ist. Komponenten enthalten Calculator, Kalender, Cardfile, Clipboard Viewer, Uhr, Control Panel, Notepad, Paint, Reversi, Terminal und Schreiben. Windows 1.0 erlaubt keine überlappenden Fenster. Stattdessen sind alle Fenster gefliest. Nur Modal Dialogboxen können über andere Fenster erscheinen. Windows 2.1 wurde in zwei verschiedenen Versionen veröffentlicht: Windows/286 und Windows/386.Windows/386 verwendet den virtuellen 8086-Modus des Intel 80386, um mehrere DOS-Programme zu multiplizieren und das paged-Speichermodell erweiterte Speicher mit verfügbarem erweiterten Speicher emulieren. Windows/286, trotz seines Namens, läuft auf beiden Intel 8086 und Intel 80286 Prozessoren. Es läuft im realen Modus, kann aber den hohen Speicherbereich nutzen. Neben vollen Windows-Paketen gab es nur Runtime-Versionen, die mit der frühen Windows-Software von Dritten versendet wurden und es ermöglichten, ihre Windows-Software auf MS-DOS und ohne das volle Windows-Funktionsset auszuführen. Die frühen Versionen von Windows werden oft als grafische Shells gedacht, vor allem weil sie auf MS-DOS lief und verwenden es für Dateisystem-Dienste. Doch auch die frühesten Windows-Versionen nahmen bereits viele typische Betriebssystemfunktionen an; insbesondere mit einem eigenen ausführbaren Dateiformat und mit eigenen Gerätetreibern (Treiber, Grafik, Drucker, Maus, Tastatur und Ton). Im Gegensatz zu MS-DOS erlaubt Windows Benutzern, mehrere grafische Anwendungen gleichzeitig durch kooperative Multitasking auszuführen. Windows implementierte ein aufwändiges, segmentbasiertes, software-virtuelles Speicherprogramm, das es ermöglicht, Anwendungen größer als verfügbarer Speicher auszuführen: Codesegmente und Ressourcen werden eingetauscht und weggeworfen, wenn der Speicher knapp wurde; Datensegmente bewegten sich im Speicher, wenn eine bestimmte Anwendung aufgegeben Prozessorsteuerung. Windows NT Versionshistorie Frühe Versionen (Windows NT) 3.1/3.5/3.51/4.0/2000) Im November 1988 startete ein neues Entwicklungsteam in Microsoft (das ehemalige Entwickler der Digital Equipment Corporation Dave Cutler und Mark Lucovsky umfasste) an einer überarbeiteten Version des Betriebssystems OS/2 von IBM und Microsoft, bekannt als "NT OS/2". NT OS/2 sollte ein sicheres, Multi-User-Betriebssystem mit POSIX-Kompatibilität und ein modularer, tragbarer Kern mit präemptivem Multitasking und Unterstützung für mehrere Prozessorarchitekturen sein. Nach der erfolgreichen Veröffentlichung von Windows 3.0 entschied sich das NT-Entwicklungsteam jedoch, das Projekt neu zu bearbeiten, um einen erweiterten 32-Bit-Port der Windows-API als Win32 anstelle von denen von OS/2 zu verwenden. Win32 pflegte eine ähnliche Struktur wie die Windows-APIs (ermöglichte die einfache Portierung bestehender Windows-Anwendungen auf die Plattform), unterstützte aber auch die Fähigkeiten des vorhandenen NT-Kernels. Nach seiner Zustimmung durch Microsofts Mitarbeiter, Entwicklung fortgesetzt auf, was jetzt Windows NT, die erste 32-Bit-Version von Windows. IBM widersprach jedoch den Veränderungen und setzte schließlich die OS/2-Entwicklung alleine fort. Windows NT war das erste Windows-Betriebssystem basierend auf einem Hybrid-Kernel. Der Hybrid-Kernel wurde als modifizierter Mikrokernel konzipiert, der von Richard Rashid an der Carnegie Mellon University entwickelt wurde, ohne jedoch alle Kriterien eines reinen Mikrokerns zu erfüllen. Die erste Veröffentlichung des resultierenden Betriebssystems, Windows NT 3.1 (bezeichnet, um es mit Windows 3.1 zu verbinden) wurde im Juli 1993 veröffentlicht, mit Versionen für Desktop-Workstations und Server. Mehrsprachige Unterstützung Mehrsprachige Unterstützung wurde in Windows seit Windows 3.0 gebaut. Die Sprache sowohl für die Tastatur als auch für die Schnittstelle kann über die Region und die Sprachsteuerung geändert werden. Komponenten für alle unterstützten Eingabesprachen, wie Input Method Editors, werden automatisch während der Windows-Installation installiert (in Windows XP und früher, Dateien für ostasiatische Sprachen, wie Chinesisch, und rechts-links-Skripte, wie Arabisch, müssen separat installiert werden, auch von der genannten Systemsteuerung). Drittanbieter-IMEs können auch installiert werden, wenn ein Benutzer das Gefühl hat, dass die bereitgestellte für ihre Bedürfnisse unzureichend ist. Schnittstellensprachen für das Betriebssystem sind kostenlos zum Download, aber einige Sprachen sind auf bestimmte Ausgaben von Windows beschränkt. Language Interface Packs (LIPs) sind wiederverteilbar und können von Microsofts Download Center heruntergeladen und für jede Ausgabe von Windows (XP oder später) installiert werden – sie übersetzen die meisten, aber nicht alle, der Windows-Schnittstelle und benötigen eine bestimmte Basissprache (die Sprache, die ursprünglich von Windows ausgeliefert wird). Dies wird für die meisten Sprachen in Schwellenländern verwendet. Full Language Packs, die das komplette Betriebssystem übersetzt, sind nur für bestimmte Ausgaben von Windows (Ultimate und Enterprise Editionen von Windows Vista und 7 und alle Ausgaben von Windows 8, 8.1 und RT außer Single Language) verfügbar. Sie benötigen keine spezifische Basissprache und werden häufig für mehr populäre Sprachen wie Französisch oder Chinesisch verwendet. Diese Sprachen können nicht über das Download Center heruntergeladen werden, sondern als optionale Updates über den Windows Update Service (außer Windows 8). Die Schnittstellensprache der installierten Anwendungen wird nicht durch Änderungen in der Windows-Schnittstellensprache beeinflusst. Die Verfügbarkeit von Sprachen hängt von den Anwendungsentwicklern selbst ab. Windows 8 und Windows Server 2012 stellen ein neues Sprachsteuerungs-Panel vor, in dem sowohl die Schnittstellen- als auch Eingabesprachen gleichzeitig geändert werden können, und Sprachpakete, unabhängig vom Typ, können von einem zentralen Standort heruntergeladen werden. Die PC-Einstellungen-App in Windows 8.1 und Windows Server 2012 R2 enthält dazu auch eine Gegeneinstellungsseite. Ändern der Schnittstellensprache ändert auch die Sprache der vorinstallierten Windows Store-Apps (wie Mail, Maps und News) und bestimmter anderer Microsoft-entwickelter Apps (wie Remote Desktop). Die oben genannten Einschränkungen für Sprachpakete sind jedoch noch in Kraft, außer dass für jede Ausgabe mit Ausnahme von Single Language Vollsprachenpakete installiert werden können, die auf aufstrebende Märkte ausgerichtet sind. Während die Windows 9x-Serie die Möglichkeit bot, Profile für mehrere Benutzer zu haben, hatten sie kein Konzept der Zugriffsrechte und erlaubten nicht gleichzeitigen Zugriff; und so waren nicht echte Multi-User-Betriebssysteme. Zudem realisierten sie nur einen Teilspeicherschutz. Sie wurden daher wegen mangelnder Sicherheit weitgehend kritisiert. Die Windows NT-Serie von Betriebssystemen hingegen sind echte Multi-User und implementieren absoluten Speicherschutz. Viele der Vorteile, ein echtes Multi-User-Betriebssystem zu sein, wurden jedoch dadurch behoben, dass vor Windows Vista das erste Benutzerkonto, das während des Setup-Prozesses erstellt wurde, ein Administratorkonto war, das auch der Standard für neue Konten war. Wie Windows XP hatte begrenzte Konten, die meisten Heimnutzer änderten sich nicht auf einen Kontotyp mit weniger Rechten – teilweise aufgrund der Anzahl der Programme, die unnötig Administratorrechte erforderten – und so liefen die meisten Heimnutzer immer als Administrator. Windows Vista ändert dies durch die Einführung eines Privileg Elevation System namens User Account Control. Bei der Anmeldung als Standardbenutzer wird eine Anmeldesitzung erstellt und ein Token mit nur den grundlegendsten Privilegien zugewiesen. Auf diese Weise ist die neue Anmeldungssitzung nicht in der Lage, Änderungen vorzunehmen, die das gesamte System beeinflussen würden. Beim Einloggen als Benutzer in der Gruppe Administratoren werden zwei separate Token zugeordnet. Das erste Zeichen enthält alle Privilegien, die typischerweise an einen Administrator vergeben werden, und das zweite ist ein eingeschränktes Zeichen ähnlich dem, was ein Standard-Benutzer erhalten würde. Benutzeranwendungen, einschließlich der Windows-Shell, werden dann mit dem eingeschränkten Token gestartet, was zu einer reduzierten privilegierten Umgebung auch unter einem Administrator-Konto führt. Wenn eine Anwendung höhere Privilegien fordert oder "Run as Administrator" geklickt wird, wird UAC zur Bestätigung aufgefordert und, wenn eine Einwilligung erteilt wird (einschließlich Administrator-Anmeldeinformationen, wenn das Konto, das die Erhebung anfordert, nicht Mitglied der Administratorgruppe ist), den Prozess mit dem unbeschränkten Zeichen starten. Dateiberechtigungen Alle Windows-Versionen von Windows NT 3 basieren auf einem Dateisystemberechtigungssystem, das als AGDLP (Accounts, Global, Domain Local, Berechtigungen) bezeichnet wird, in dem Dateiberechtigungen in Form einer "lokalen Gruppe", die dann andere "globale Gruppen" als Mitglieder hat, auf die Datei/Ordner angewendet werden. Diese globalen Gruppen halten dann andere Gruppen oder Benutzer abhängig von verschiedenen verwendeten Windows-Versionen. Dieses System unterscheidet sich von anderen Anbieter-Produkten wie Linux und NetWare, da die statische Zuordnung der Berechtigung direkt auf die Datei oder den Ordner angewendet wird. Mit diesem Prozess der AGLP/AGDLP/AGUDLP lässt sich jedoch eine kleine Anzahl von statischen Berechtigungen anwenden und ermöglicht einfache Änderungen an den Kontogruppen, ohne die Dateiberechtigungen auf den Dateien und Ordnern neu zu nutzen. Dazu gehören: Wein – eine kostenlose und Open-Source-Implementierung der Windows-API, so dass man viele Windows-Anwendungen auf x86-basierten Plattformen, darunter UNIX, Linux und macOS, ausführen kann. Weinentwickler beziehen sich darauf als "Kompatibilitätsschicht" und verwenden Windows-Stil APIs, um Windows-Umgebung zu emulieren. CrossOver – ein Weinpaket mit lizenzierten Schriften. Seine Entwickler sind regelmäßige Mitarbeiter an Wine und konzentrieren sich auf offiziell unterstützte Anwendungen. Cedega – eine proprietäre Gabel von Wein von TransGaming Technologies, speziell für die Nutzung von Microsoft Windows-Spielen auf Linux entwickelt. Da Wein unter der LGPL lizenziert wurde, ist Cedega nicht in der Lage, die Verbesserungen an Wein an ihre proprietäre Codebasis zu portieren. Cedega hat seinen Dienst im Februar 2011 eingestellt. Darwine – ein Hafen von Wein für macOS und Darwin. Operiert mit Wein auf der QEMU.
