Cloud Foundry ist eine Open Source, Multi-Cloud-Anwendungsplattform als Service (PaaS) der Cloud Foundry Foundation, eine 501(c)(6 Organisation. Die Software wurde ursprünglich von VMware entwickelt, die auf Pivotal Software übertragen wurde (ein Joint Venture von EMC, VMware und General Electric), die dann die Software auf die Cloud-Gründer-Stiftung übertragen hat. Geschichte Ursprünglich 2009 konzipiert und entwickelt wurde Cloud Foundry von einem kleinen Team von VMware unter der Leitung von Derek Collison und wurde ursprünglich Project B29 genannt. Ein anderes in Java für Amazon EC2 geschriebenes PaaS-Projekt nutzte damals den Namen Cloud Foundry. Es wurde 2008 von Chris Richardson gegründet und 2009 von SpringSource erworben, im selben Jahr erwarb VMware SpringSource. Das aktuelle Projekt steht nicht im Zusammenhang mit dem Projekt unter SpringSource, aber der Name wurde angenommen, als das ursprüngliche SpringSource-Projekt beendet wurde. Die Ankündigung der Cloud Foundry fand im April 2011 statt. Ein Jahr später, im April 2012, wurde BOSH, eine Open-Source-Tool-Kette für Release Engineering, Bereitstellung und Life-Cycle-Management von großen verteilten Dienstleistungen, öffentlich gestartet. Im April 2013 wurde Pivotal von EMC und VMware geschaffen, um Assets wie Cloud Foundry, RabbitMQ und Spring zu vermarkten. Bis Februar 2014 wurde bekannt gegeben, dass es eine offene Governance-Stiftung mit sieben Platinum-Mitgliedern und zwei Gold-Mitgliedern gäbe. Im Mai 2014 gab es eine Ankündigung einer erweiterten Mitgliedschaft mit acht neuen Unternehmen. Bis Dezember 2014 stieg die Mitgliedschaft auf 40. Cloud Foundry FoundationIm Januar 2015 wurde die Cloud Foundry Foundation als unabhängiges Non-Profit 501(c)(6)Linux Foundation Collaborative Project gegründet. Nach der Gründung der Cloud Foundry Foundation wurde die Cloud Foundry Software (Quellencode und alle zugehörigen Marken) von der Open Source Software Foundation übertragen. Es wird hauptsächlich in Ruby, Go und Java geschrieben. Ab Februar 2019 hatte die Stiftung 65 Mitglieder. Die Stiftung dient als neutrale Partei, die alle intellektuellen Immobilien der Cloud-Gießerei hält. Sobald geistiges Eigentum zu Cloud-Gründer beigetragen wird, wird es Eigentum der Stiftung, und diese Vermögenswerte können nicht an gemeinnützige Organisationen übertragen werden, nach den Gesetzen für 501(c)(6 Organisationen. Die Stiftung verfügt über zwei Beiträge zum geistigen Eigentum: Marken und Urheberrechte an der kollektiven Arbeit der Gemeinschaft. Es hat auch eine Lizenz, alle Codebeiträge zu verwenden und neu zu lizenzieren, aber keine eigenen Urheberrechte auf diesen Beiträgen. Die Verwendung von Cloud-Gründer wird für die kontinuierliche Lieferung gefördert, da sie den gesamten Anwendungsentwicklungs-Lebenszyklus unterstützt, von der ersten Entwicklung über alle Testschritte bis zur Bereitstellung. Cloud Foundrys containerbasierte Architektur betreibt Apps in jeder Programmiersprache über eine Vielzahl von Cloud-Dienstleistern. Diese Multi-Cloud-Umgebung ermöglicht es Entwicklern, die Cloud-Plattform zu verwenden, die bestimmten Anwendungs-Workloads entspricht und diese Workloads innerhalb von Minuten ohne Änderungen an der Anwendung möglichst schnell verschieben kann. Services Applikationen, die Cloud-Gründer auf externe Ressourcen zugreifen, über eine Open Service Broker API, die im Dezember 2016 gestartet wurde. In einer Plattform werden alle externen Abhängigkeiten wie Datenbanken, Messaging-Systeme, Dateisysteme usw. als Dienste betrachtet. Cloud Foundry ermöglicht es Administratoren, einen Marktplatz von Dienstleistungen zu schaffen, von denen Nutzer diese Dienste auf Anfrage bereitstellen können. Wenn eine Anwendung auf Cloud-Gründer verschoben wird, können die von ihr benötigten Dienste auch angegeben werden. Dieser Prozess bringt die Anmeldeinformationen in eine Umgebungsvariable. SoftwareDie Entwicklung von Cloud Foundry wird durch den von seinen Mitgliedern vereinbarten Governance-Prozess von der Cloud Foundry Foundation unterstützt. Der Quellcode ist unter einer Apache-Lizenz 2.0.und Beiträge werden auf Basis der Cloud-Gründer-Lizenzen für Einzelpersonen und Unternehmen erstellt. Cloud Foundry ist in der Lage, durch Zusammenarbeit mit anderen Projekten und Standards wie Open Container Initiative (OCI) und Container Network Interface (CNI) Innovationen in Containern zu unterstützen. Sprachen und Frameworks, die mit dem Buildpack Lebenszyklus eingesetzt werden können, umfassen: Neben dem Buildpack-Lebenszyklus können Anwendungen, die als Docker-Bilder verpackt sind, mit dem Docker-Lebenszyklus eingesetzt werden. Cloud Foundry Application Runtime kann für viele verschiedene Infrastrukturanbieter eingesetzt werden, darunter VMwares vSphere, OpenStack, Amazon Web Services, Microsoft Azure, IBM Cloud, Google Cloud Platform, SAP Cloud Platform, Alibaba Cloud und andere über die Cloud Provider Interface (CPI)-Fähigkeit des Cloud Foundry BOSH-Projekts.Es ist auch möglich, Cloud Foundry mit einer virtuellen BOSH-Lite Vagrant-Maschine zu installieren. Plattform Die Cloud Foundry-Plattform ist entweder von der Cloud Foundry Foundation als Open-Source-Software oder von einer Vielzahl von kommerziellen Anbietern als Softwareprodukt erhältlich oder als Service erbracht. Cloud Foundry ist Open-Source-Software, und daher ist es jedem zur Verfügung. Die Bereitstellung von Cloud-Gründer umfasst die Verknüpfung mit der zugrunde liegenden Infrastruktur mit dem Cloud-Gründer BOSH-Bereitstellungssystem, einem weiteren Open-Source-Tool der Cloud-Gründer-Stiftung. Im Dezember 2015 kündigte die Cloud-Gründer-Stiftung das „Cloud-Gründer-PaaS-Zertifizierungsprogramm“ an, das Kriterien als Cloud-Gründer-zertifizierter Provider definierte.
