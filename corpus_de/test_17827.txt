Algorand ist eine Blockchain-basierte Kryptowährungsplattform, die sicher, skalierbar und dezentral sein soll. Die Algorand-Plattform unterstützt intelligente Vertragsfunktionalität, und ihr Konsensalgorithmus basiert auf Nachweisprinzipien und einem byzantinischen Vertragsprotokoll. Algorands native Kryptowährung wird Algo genannt. Geschichte und Entwicklung Die Entwicklung der Algorand-Plattform wird von Algorand, Inc., einem privaten Unternehmen mit Sitz in Boston, übertroffen. Es wurde 2017 von Silvio Micali, Professor am MIT, gegründet. Das Algorand-Testnetzwerk wurde im April 2019 öffentlich gestartet und das Hauptnetz wurde im Juni 2019 gestartet. Design Consensus Algorithmus Im Algorand-Netzwerk ist der Konsensalgorithmus berechtigt, und alle Benutzer, die eine Algo-Balance halten, können teilnehmen. Der Konsensusalgorithmus arbeitet in Runden, wobei jede Runde aus zwei Phasen besteht. Die erste Phase ist die Blockvorschlagsphase, in der Blöcke als neuer Block vorgeschlagen werden; die zweite Phase ist die Blockendisierungsphase, bei der eine Abstimmung über die vorgeschlagenen Blöcke erfolgt. Die erste Phase (die Blockvorschlagsphase) verwendet den Nachweis von Stake-Prinzipien. Während dieser Phase wird ein Benutzerausschuss im System zufällig ausgewählt, obwohl in einer gewichteten Weise, um den neuen Block vorzuschlagen. Die Auswahl des Ausschusses erfolgt über einen Prozess mit dem Namen „cryptographic sortition“. Bei der kryptographischen Sortierung gibt es keine zentrale Behörde, die angibt, wer die Mitglieder des Ausschusses sind und dann diese Informationen über das Netzwerk kommuniziert; vielmehr bestimmt jeder Benutzer, ob er im Ausschuss ist oder nicht, indem er eine vertretbare Random-Funktion (VRF) lokal ausführt. Wenn der VRF angibt, dass der Benutzer gewählt wird, gibt der VRF einen kryptographischen Beweis zurück, der verwendet werden kann, um zu überprüfen, ob der Benutzer im Ausschuss ist. Nur ein bestimmter Benutzer weiß, ob sie im Ausschuss sind, es sei denn/bis sie eine Nachricht an andere Benutzer senden, die angeben, dass sie sind. Die Wahrscheinlichkeit, dass ein bestimmter Benutzer im Ausschuss sein wird, wird durch die “Stake” (d.h. die Anzahl der Algo-Tokens), die von diesem Benutzer gehalten wird, im Verhältnis zur Größe des Einsatzes des Benutzers beeinflusst. Nach der Feststellung, dass sie sich im Blockauswahlausschuss befinden, baut ein Benutzer einen vorgeschlagenen Block und gibt es in der zweiten Phase in das Netz zur Überprüfung/Analyse. Der Benutzer enthält den kryptographischen Nachweis der VRF in ihrem vorgeschlagenen Block, der zeigt, dass der Benutzer tatsächlich ein förderfähiges Komiteemitglied war. In der zweiten Phase (der Blockfinalisationsphase) wird ein Byzantinisches Vertragsprotokoll (im Folgenden „BA-Freizeit“) zur Abstimmung über die vorgeschlagenen Blöcke verwendet. In dieser zweiten Phase wird die vorbeschriebene kryptographische Sortierung erneut zur Ermittlung eines Ausschusses verwendet; dieser Zweiphasen-Abstimmungsausschuss wird sich vom Ausschuss aus der ersten Phase unterscheiden, obwohl es möglich ist, dass sich die Mitgliedschaft zwischen den beiden Ausschüssen überschneiden könnte. Wenn die Nutzer festgestellt haben, dass sie in diesem zweiten Wahlausschuss sind, analysieren sie die vorgeschlagenen Blöcke, die sie erhalten haben (dies beinhaltet die Überprüfung, dass sie tatsächlich von den Benutzern aus dem ersten Phase Ausschuss vorgeschlagen wurden) und wählen, ob eine der Blöcke angenommen werden sollte oder nicht. Erreicht der Abstimmungsausschuss Konsens über einen neuen Block, so wird der neue Block als neuer Block über das Netzwerk verbreitet. Der Algorand-Konsensationsalgorithmus besitzt die Charakteristik der "Spieleraustauschbarkeit", d.h., wie oben erwähnt, ändert sich die Mitgliedschaft in den verschiedenen Ausschüssen (in der Blockvorschlag- und Blockabschlussphase) jedes Mal, wenn die Phase ausgeführt wird. Dies schützt die Nutzer vor gezielten Angriffen, da ein Angreifer nicht im Voraus wissen wird, welche Nutzer in einem Ausschuss sein werden. Algorand ist gegen beliebige Trennwände, auch als asynchrone Sicherheit bekannt, federnd. Zwei verschiedene Blöcke können in der gleichen Runde keinen Konsens erreichen, d.h. es ist mathematisch garantiert, dass Algorand nicht verlassen wird. Die asynchrone Sicherheit wurde auch von Runtime Verification Inc. formal überprüft und im Vergleich zu ihren bisherigen Verifikationsmodellen, das Modell auch für Timing-Probleme und Adversary-Aktionen, z.B. wenn der Gegner die Kontrolle über die Nachrichtenlieferung hat. Smart Contracts Algorand unterstützt zwei Arten von Smart Contracts: Stateless Smart Contracts und Stateful Smart Contracts. Staatslose Smart Contracts sind zum Zwecke der Autorisierung von Transaktionen bestimmt; Stateful Smart Contracts speichern Daten persistent und können für breitere Zwecke verwendet werden. Algorand Smart Contracts können in einer Programmiersprache namens Transaction Execution Approval Language (TEAL) geschrieben werden.TEAL ist eine bytecode-basierte Stack-Sprache, mit einer Programmierschnittstelle für Python, die PyTeal heißt. Während einige intelligente Vertragsprogrammierungsmodelle Turing-complete sind (z.B. Solidity ist Turing-complete), ist das Algorand Smart Contracts Modell nicht Turing-complete. Das Algorand Smart Contracts-Modell unterstützt Transaktionsatome. In einigen anderen Blockchain-Systemen werden Smart Contracts verwendet, um benutzerdefinierte Assets zu definieren; zum Beispiel in Ethereum implementieren Smart Contracts die Schnittstellen ERC20 und ERC721, um neue Assets zu definieren. In Algorand hingegen werden benutzerdefinierte Aktiva in nativer Form unterstützt, und Algorand Smart Contracts sind in der Lage, benutzerdefinierte Assets zu manipulieren (z.B. durch die Übertragung von Eigentum an bestimmten Mengen von ihnen) mit integrierten Transaktionstypen. Im März 2021 hat die Società Italiana degli Autori ed Editori (SIAE) die italienische Urheberrechtssammlungsstelle, die über vier Millionen nicht-fungible Token (NFTs) an die Algorand-Blockchain hochgeladen hat und Urheberrechte in Werken von SIAE-Mitgliedern darstellt. Im Februar 2021 kündigte Ditto Music die Einführung eines Projekts auf dem Algorand-Netzwerk namens Opulous an; Opulous wird ein dezentraler Finanzierungspool (DeFi) sein, in dem Kredite gegen die früheren Streaming-Einnahmen und die Urheberrechte der Künstler als Sicherheit gehalten werden. Im Juni 2020 veröffentlichte die Republik ein Unternehmen, das Crowdfunding-Kampagnen für Start-ups und kleine bis mittlere Unternehmen (SMBs) ermöglichte, einen Gewinnbeteiligungs-Token auf der Algorand-Plattform. Im Februar 2020 kündigte PlanetWatch, ein Spinoff von CERN, ein Programm an, das die Bereitstellung eines globalen Netzwerks von Luftqualitätssensoren umfasst, in dem die Sensoren Messdaten auf die Algorand-Blockchain aufnehmen.
