Eine Rechenplattform oder digitale Plattform ist eine Umgebung, in der ein Stück Software ausgeführt wird. Es kann die Hardware oder das Betriebssystem (OS), auch ein Webbrowser und zugehörige Anwendungsprogrammierungsschnittstellen oder eine andere zugrunde liegende Software sein, solange der Programmcode mit ihm ausgeführt wird. Computing-Plattformen haben verschiedene Abstraktionsebenen, darunter eine Computerarchitektur, ein Betriebssystem oder Laufzeitbibliotheken. Eine Rechenplattform ist die Stufe, auf der Computerprogramme laufen können. Eine Plattform kann sowohl als Einschränkung auf dem Software-Entwicklungsprozess betrachtet werden, indem verschiedene Plattformen unterschiedliche Funktionalitäten und Einschränkungen bieten und als Assistent des Entwicklungsprozesses, indem sie Low-Level-Funktionalität bereit stellen. Ein OS kann beispielsweise eine Plattform sein, die die zugrunde liegenden Unterschiede in der Hardware abstrahiert und einen generischen Befehl zum Speichern von Dateien oder zum Zugriff auf das Netzwerk bereitstellt. Komponenten Plattformen können auch umfassen: Hardware allein, bei kleinen eingebetteten Systemen. Embedded-Systeme können auf Hardware direkt zugreifen, ohne ein Betriebssystem; dies wird als Laufen auf "bares Metall" bezeichnet. Ein Browser bei webbasierter Software. Der Browser selbst läuft auf einer Hardware+OS-Plattform, aber dies ist für Software, die innerhalb des Browsers läuft, nicht relevant. Eine Anwendung, wie ein Tabellenkalkulations- oder Wortprozessor, der Software in einer anwendungsspezifischen Skriptsprache, wie einem Excel-Makro, hostet. Dies kann auf vollwertige Anwendungen mit der Microsoft Office Suite als Plattform erweitert werden. Software-Frameworks, die eine fertige Funktionalität bieten. Cloud Computing und Platform als Service. Erweitern Sie die Idee eines Software-Frameworks, ermöglichen diese Anwendungsentwickler Software aus Komponenten, die nicht vom Entwickler, sondern vom Anbieter gehostet werden, mit Internet-Kommunikation, die sie miteinander verknüpft. Die Social-Networking-Sites Twitter und Facebook gelten auch als Entwicklungsplattformen. Eine virtuelle Maschine (VM) wie die Java virtuelle Maschine oder NETCLR. Anwendungen werden in ein dem Maschinencode ähnliches Format zusammengestellt, das als Bytecode bekannt ist, das dann vom VM ausgeführt wird. Eine virtualisierte Version eines kompletten Systems, einschließlich virtualisierter Hardware, OS, Software und Storage. Diese erlauben zum Beispiel ein typisches Windows-Programm, auf dem zu laufen, was physisch ein Mac ist. Einige Architekturen haben mehrere Schichten, mit jeder Schicht als Plattform zu der darüber. Im allgemeinen muss ein Bauteil nur unmittelbar darunter an die Schicht angepasst werden. So muss beispielsweise ein Java-Programm geschrieben werden, um die Java-Virtual-Maschine (JVM) und zugehörige Bibliotheken als Plattform zu verwenden, muss aber nicht an die Windows-, Linux- oder Macintosh-Betriebsplattformen angepasst werden. Allerdings muss die JVM, die Schicht unter der Anwendung, für jedes OS separat gebaut werden.
