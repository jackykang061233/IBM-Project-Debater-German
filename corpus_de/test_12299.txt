Eucalyptus ist eine kostenpflichtige und Open-Source-Computersoftware für den Aufbau von Amazon Web Services (AWS)-kompatiblen privaten und hybriden Cloud Computing-Umgebungen, die ursprünglich von der Firma Eucalyptus Systems entwickelt wurden. Eucalyptus ist ein Akronym für Elastische Utility Computing Architecture zur Verknüpfung Ihrer Programme zu Nützlichen Systemen. Eucalyptus ermöglicht die Bündelung von Rechen-, Speicher- und Netzwerkressourcen, die bei Änderung der Anwendungs-Workloads dynamisch nach oben oder unten skaliert werden können. Eucalyptus verwendet die Terminologie: Bilder – Ein Bild ist eine feste Sammlung von Software-Modulen, Systemsoftware, Anwendungssoftware und Konfigurationsinformationen, die von einer bekannten Basislinie gestartet wird (unmutbar/fixiert). Wenn gebündelt und in die Eucalyptus-Cloud hochgeladen wird, wird dies ein Eucalyptus-Maschinenbild (EMI). Instanzen – Wenn ein Bild verwendet wird, wird es als Instanz bezeichnet. Die Konfiguration wird zur Laufzeit ausgeführt, und der Cloud-Controller entscheidet, wo das Bild ausgeführt wird, und die Speicherung und Vernetzung wird an den Ressourcenanforderungen angepasst. IP-Adresse – Eucalyptus-Instanzen können öffentliche und private IP-Adressen haben. Eine IP-Adresse wird einer Instanz zugeordnet, wenn die Instanz aus einem Bild erzeugt wird. Beispielsweise, die eine persistente IP-Adresse benötigen, wie z.B. einen Webserver, liefert Eucalyptus elastische IP-Adressen. Diese werden durch die Eucalyptus-Cloud vorversetzt und können einer laufenden Instanz wieder zugeordnet werden. Sicherheit – TCP/IP Sicherheitsgruppen teilen eine gemeinsame Reihe von Firewall-Regeln. Dies ist ein Mechanismus, um eine Instanz mit IP-Adresse und Port-Block/Allow-Funktionalität zu löschen. Instanzen werden bei TCP/IP-Schicht isoliert 2. Wäre dies nicht vorhanden, könnte ein Benutzer die Vernetzung von Instanzen manipulieren und Zugriff auf benachbarte Instanzen gewinnen, die den Grundwolkentakt der Instanztrennung und -trennung verletzen. Networking – Es gibt drei Netzwerk-Modi. Im Managed Mode verwaltet Eucalyptus ein lokales Netzwerk von Instanzen, darunter Sicherheitsgruppen und IP-Adressen. Im Systemmodus ordnet Eucalyptus eine MAC-Adresse zu und legt die Netzwerkschnittstelle der Instanz über die Brücke des Node Controllers an das physikalische Netzwerk an. Der Systemmodus bietet keine elastischen IP-Adressen, Sicherheitsgruppen oder VM-Isolierung an. Im statischen Modus ordnet Eucalyptus IP-Adressen an Instanzen zu. Statische Mode bietet keine elastischen IPs, Sicherheitsgruppen oder VM Isolation. Access Control – Ein Benutzer von Eucalyptus ist einer Identität zugeordnet, und Identitäten können für die Zugriffskontrolle gemeinsam gruppiert werden. Komponenten Eucalyptus hat sechs Komponenten: Der Cloud Controller (CLC) ist ein Java-Programm, das EC2-kompatible Schnittstellen bietet, sowie eine Web-Schnittstelle zur Außenwelt.Neben eingehenden Anfragen fungiert der CLC als administrative Schnittstelle für das Cloud-Management und führt hochrangige Ressourcenplanung und Systemrechnung durch. Der CLC akzeptiert Benutzer-API-Anfragen von Befehlszeilenschnittstellen wie euca2ools oder GUI-basierten Tools wie der Eucalyptus User Console und verwaltet die zugrunde liegenden Rechen-, Speicher- und Netzwerkressourcen. Nur ein CLC kann pro Cloud existieren und behandelt Authentifizierung, Rechnungslegung, Berichterstattung und Quotenmanagement. Walrus, auch in Java geschrieben, ist der Eucalyptus gleichwertig mit AWS Simple Storage Service (S3). Walrus bietet dauerhafte Speicherung an alle virtuellen Maschinen in der Eucalyptus-Cloud und kann als einfache HTTP Put/get-Speicher als Servicelösung verwendet werden. Es gibt keine Einschränkungen für den Datentyp für Walrus, und es kann Bilder (d.h. die Bausteine verwendet, um virtuelle Maschinen zu starten), Volumen-Schnappshots (d.h. Punkt-in-time Kopien,) und Anwendungsdaten enthalten. Nur ein Walrus kann pro Wolke existieren. Der Cluster Controller (CC) wird in C geschrieben und fungiert als vorderes Ende für einen Cluster innerhalb einer Eucalyptus-Cloud und kommuniziert mit dem Storage Controller und Node Controller. Es verwaltet Instanz (d.h. virtuelle Maschinen) Ausführung und Service Level Agreements (SLAs) pro Cluster. Der Storage Controller (SC) ist in Java geschrieben und entspricht dem Eucalyptus AWS EBS. Es kommuniziert mit dem Cluster Controller und Node Controller und verwaltet Eucalyptus-Blockvolumen und Snapshots zu den Instanzen innerhalb seines spezifischen Clusters. Wenn eine Instanz das Schreiben von persistenten Daten außerhalb des Clusters erfordert, müsste sie an Walrus schreiben, die in jedem Cluster jeder Instanz zur Verfügung steht. Der VMware Broker ist eine optionale Komponente, die eine AWS-kompatible Schnittstelle für VMware-Umgebungen bietet und physisch auf dem Cluster Controller läuft. Der VMware Broker überlagert bestehende ESX/ESXi-Hosts und verwandelt Eucalyptus Machine Images (EMIs) auf VMware virtuelle Festplatten. Der VMware Broker vermittelt Interaktionen zwischen dem Cluster Controller und VMware und kann direkt an ESX/ESXi-Hosts oder an vCenter Server angeschlossen werden. Der Node Controller (NC) wird in C geschrieben und beherbergt die virtuellen Maschineninstanzen und verwaltet die virtuellen Netzwerkendpunkte. Es Downloads und Caches Bilder von Walrus sowie erstellt und Caches Instanzen. Zwar gibt es keine theoretische Grenze zur Anzahl der Node-Controller pro Cluster, doch gibt es Leistungsgrenzen. Im Juni 2013 wurde Eucalyptus 3.3 veröffentlicht, mit einer neuen Serie von AWS-kompatiblen Werkzeugen. Dazu gehören: Autoscaling – Ermöglicht Anwendungsentwickler, die Eucalyptus-Cloud-Ressourcen nach oben oder unten zu skalieren, um die Leistung zu erhalten und SLAs zu treffen. Mit der Auto-Skalierung können Entwickler Instanzen und virtuelle Maschinen hinzufügen, da sich die Verkehrsanforderungen erhöhen. Auto-Skalierung Richtlinien für Eucalyptus werden mit Amazon EC2-kompatiblen APIs und Tools definiert. Elastic Load Balancing – Ein Service, der eingehende Anwendungs- und Serviceanrufe über mehrere Eucalyptus-Workload-Instanzen verteilt und eine größere Anwendungsfehlertoleranz bietet. CloudWatch – Ein Überwachungstool ähnlich Amazon CloudWatch, das Ressourcen und Anwendungen auf Eucalyptus Clouds überwacht. Mit CloudWatch können Anwendungsentwickler und Cloud-Administratoren die Erfassung von Metriken, Alarme einstellen und Trends identifizieren, die die Workload-Operationen gefährden, und Maßnahmen ergreifen, um sicherzustellen, dass ihre Anwendungen weiterhin reibungslos laufen. Eucalyptus 3.3 ist auch die erste private Cloud-Plattform, um Netflixs Open Source-Tools – einschließlich Chaos Monkey, Asgard und Edda – durch seine API-Fidelity mit AWS zu unterstützen.
